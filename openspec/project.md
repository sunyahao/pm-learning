# 项目上下文

## 项目名称

gaoxiang-spec

## 项目目标

构建并维护一个基于 OpenSpec 管理的高项备考仓库，使章节学习资产既能直接用于复习，也能严格对齐来源 PDF 的目录和内容边界，并在时间受限条件下尽可能提高拿证通过率。

## 主来源材料

- `高项-三色笔记.pdf`

## 仓库边界

- `openspec/`：保存 OpenSpec 配置、当前规范和归档的变更历史。
- `chapters/`：保存直接面向复习的派生学习资产，每章使用独立目录组织。
- `guides/`：保存总览型备考策略、真题趋势总结和冲刺计划。
- `README.md`：给人看的仓库入口说明。
- `AGENTS.md`：给依赖 AGENTS 约定的 AI 工具看的根部协作说明。

## 领域模型

1. `高项-三色笔记.pdf` 是章节学习资产的主来源。
2. 论文、案例和选择题历年真题是题型趋势分析、答题训练和错因校准的正式事实来源。
3. `2.备考指导/`、`3.高频考点/`、`4.模拟试题/`、`5.优秀论文范文/`、`7.知识点汇总/`、`8.知识整理/` 等目录是讲解、押题和辅助训练来源。
4. 来源 PDF 一共有 24 章。
5. 仓库中的 `chapters/` 必须与这 24 章保持一一对应。
6. 每章使用一个目录表示，目录内至少包含 `README.md`、`主脉络.md`、`核心记忆.md` 和 `案例/`。
7. 第08章至第17章以及第19章需要额外提供 `输出物/` 目录，用于保存关键输出物样例。
8. 项目管理核心章节需要额外沉淀 ITTO 输出物速看材料。
9. 当前学习策略默认以“上班族、时间紧、目标是通过考试”为前提。
10. 每个章节 README 都要记录其 PDF 章节名和对应页码范围。
11. 真题类派生资产应明确其题型来源、年份、批次和答案依据。
12. 当前生效的仓库规则全部以 `openspec/specs/` 中的规范为准。

## 非目标

- 本仓库不是 PDF 的逐字转录版本。
- 本仓库不再使用和 PDF 目录不一致的聚合章节名。

## 编辑约束

- 任何结构性调整都应通过 OpenSpec change 描述。
- 派生章节内容应当简洁、面向考试、并能回溯到 PDF。
- 章节标题、顺序和数量应严格遵循 PDF 目录。
- 新增的备考总览类文件应放在 `guides/` 而不是混入章节目录。
- 当“内容全面”与“通过率更高”冲突时，默认优先服务高频考点、快速复习和可执行计划。
