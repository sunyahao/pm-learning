# 设计：第18章项目绩效域强化学习资产

## 设计目标

把第18章 `项目绩效域` 从“名称记忆章”提升为“能够支撑论文、案例和综合表达”的重点章节。

## 一、资产范围

第18章补齐以下四类文件：

1. `完整整理.md`
2. `精讲.md`
3. `复盘题单.md`
4. `参考作答.md`

其中：

- `完整整理.md` 负责按 PDF 主线整理八大绩效域、预期目标、度量指标、开发方法与生命周期、规划与度量要点
- `精讲.md` 负责用教学式表达讲清“绩效域不是知识域改名，而是观察项目表现的另一套视角”
- `复盘题单.md` 负责把新版高频考点转成主动回忆问题
- `参考作答.md` 负责把这些问题转成可口述、可写案例与论文的表达

## 二、学习顺序

第18章 `README.md` 的学习入口采用：

1. `完整整理.md`
2. `精讲.md`
3. `复盘题单.md`
4. `参考作答.md`
5. `主脉络.md`
6. `核心记忆.md`

核心逻辑是：

- 先把绩效域整体结构看懂
- 再听教师式串联
- 再做主动回忆
- 最后修正表达

## 三、内容重点

第18章强化内容重点围绕：

- 八大绩效域及其绩效要点
- 各绩效域预期目标、指标和检查方法
- 批判性思维、情商、决策和冲突管理
- 交付节奏、开发方法及开发方法选择
- 规划绩效域中的估算与优化
- 度量绩效域中的 KPI、展示方式和陷阱
- 与第08章至第17章传统知识域的映射关系

## 四、导航影响

补齐后需要同步更新：

- `guides/章节记忆导航.md`
- `guides/资料盘点与提分优化建议.md`

让第18章从“第二优先只看核心记忆”升级为可以系统学习的重点章节。
