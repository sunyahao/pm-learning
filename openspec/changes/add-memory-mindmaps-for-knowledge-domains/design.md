# 设计：重点知识领域重点记忆导航

## 设计目标

`重点记忆导航.md` 的目标不是替代 `核心记忆.md`，而是把本章最值得记忆的内容再压缩一层，用“重点列表 + 直接链接”的方式帮助学习者快速建立“这一章的复习导航”。

## 文档定位

适用场景：

- 章节内容较多，需要一份“总图式”记忆材料
- 学习者已经看过正式整理或精讲，希望快速收束主线
- 临考前需要在几分钟内过一遍章节结构和高频易混点

不适用场景：

- 把 `完整整理.md` 简单缩写一遍
- 脱离主来源 PDF 自由发散
- 只画流程，不体现记忆重点和高频易混点

## 推荐结构

一份 `重点记忆导航.md` 应优先包含：

1. 这份文档怎么用
2. 先背主线
3. 最该背的知识块
4. 过程或主题分组导航
5. 最易混点直接跳
6. 案例题答题抓手

补充要求：

- 每个重点条目后都应尽量给一句简明记忆点
- 关键条目应尽量链接到 `完整整理.md`、`ITTO输出速看.md`、`输出物/` 或其他对应文件
- 文档应优先服务“快速回忆 + 一键跳转”，而不是重复大段讲解

## 第08章示例范围

第08章 `整合管理` 的 `重点记忆导航.md` 应覆盖：

- 章节定位
- 三个基础抓手：整合目标、三个层面、复杂性
- 七个过程主线
- 两类高频输出与流转：章程/计划、绩效数据/报告/变更请求
- 最易混点：计划 vs 文件、配置控制 vs 变更控制、监督 vs 控制
- 答题抓手

## 学习入口顺序

如章节提供 `重点记忆导航.md`，其学习入口建议位于：

1. `主脉络.md`
2. `核心记忆.md`
3. `重点记忆导航.md`

这样能先看文字提炼，再用导航快速跳到最该背的位置。
