# 设计：知识领域白话拆解与过程化输出物目录

## 设计一：白话拆解文档扩展到十大知识领域

### 文档定位

`白话拆解.md` 继续承担“术语去抽象化”的职责，不替代：

- `完整整理.md` 的知识密度
- `精讲.md` 的系统讲解
- `过程场景讲解.md` 的业务过程主线

它只负责解决一个问题：

`明明看过概念，但放到日常项目里还是不知道它在干什么。`

### 覆盖范围

扩展到第08章至第17章，每章围绕本章最容易发虚的高频概念编写。

### 推荐结构

每份文档优先采用：

1. 这份文档解决什么问题
2. 本章最容易发虚的概念
3. 每个概念的四层解释
   - 标准说法
   - 大白话
   - 公司项目里的例子
   - 一句话记忆
4. 最容易混淆的对比
5. 放到同一个项目链路里怎么看
6. 考试里怎么用

## 设计二：核心章节输出物统一按过程目录归档

### 目录结构

第08章至第17章以及第19章的 `输出物/` 目录统一采用：

- 根目录 `README.md` 作为总索引
- 过程子目录用于存放对应样例文件

例如：

```text
输出物/
  README.md
  01-某过程/
  02-某过程/
  ...
```

### 设计原则

- 过程目录名使用当前章节标准过程或常用步骤命名
- 文件真正移动到对应子目录，不保留根目录平铺副本
- 根目录 `README.md` 负责回答“本章有哪些过程、每个过程下有哪些文件”

### 第19章处理方式

第19章不是标准 PMBOK 知识领域流程，但仍有明确的配置与变更控制步骤，因此按“配置标识、状态记录、变更申请与分析、发布与回退”等步骤归档即可。

## 学习入口顺序

第08章至第17章的 `README.md` 学习入口统一为：

1. `完整整理.md`
2. `精讲.md`
3. `过程场景讲解.md`
4. `白话拆解.md`
5. `复盘题单.md`
6. `参考作答.md`

这样先建立正式理解，再通过业务过程建立画面，再用白话拆掉术语障碍，最后进入主动回忆。
