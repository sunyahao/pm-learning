## 新增要求

### 要求：每个章节目录都要具备固定结构
每个派生章节目录都必须至少包含 `README.md`、`主脉络.md` 和 `案例/` 目录。

#### 场景：打开任一章节目录
- **前提** 读者打开 `chapters/` 下任意章节目录
- **当** 他查看目录内容
- **则** 能看到 `README.md`
- **并且** 能看到 `主脉络.md`
- **并且** 能看到 `案例/` 目录

### 要求：项目管理核心章节必须提供 ITTO 输出速看
第08章至第17章以及第19章的章节目录下必须提供 `ITTO输出速看.md`，用于快速复习关键输出物和常见文档。

#### 场景：复习项目管理过程输出物
- **前提** 读者正在复习项目管理核心章节
- **当** 他进入第08章至第17章或第19章目录
- **则** 能找到 `ITTO输出速看.md`

### 要求：每章必须预留案例归档位置
每个章节目录都必须包含 `案例/` 目录，用于放置真题、案例索引和相关材料。

#### 场景：整理章节案例
- **前提** 读者想把某章相关真题或案例材料归入对应章节
- **当** 他进入该章节目录
- **则** 能直接使用 `案例/` 目录进行整理
