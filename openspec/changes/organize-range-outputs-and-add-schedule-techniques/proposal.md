# 提案：按过程整理范围管理输出物，并为进度管理补充工具与技术样例

## 为什么要改

当前第09章 `范围管理` 已经提供了一批输出物样例，但这些文件都平铺在 `输出物/` 根目录下。学习者虽然能看到文件名，却仍然容易搞不清：

1. 这个文件到底属于哪个过程
2. `项目范围说明书`、`WBS`、`WBS词典` 在范围管理链路中的位置
3. `验收确认单` 和 `变更请求单` 为什么一个更贴近确认范围，一个更贴近控制范围

同时，第10章 `进度管理` 里已经提到 `紧前关系绘图法 PDM` 和 `箭线图法 ADM`，但仓库里还没有能直接展示“它们长什么样”的专门样例文件。学习者看到术语后，仍然容易停留在定义层面，无法形成直观印象。

## 要改什么

1. 将第09章 `输出物/` 目录按过程建立子目录，并把样例文件移动到对应过程下
2. 保留第09章 `输出物/README.md` 作为根索引，但改为按过程链接到子目录中的具体文件
3. 在第10章新增 `工具与技术/` 目录
4. 补充 `紧前关系绘图法 PDM` 与 `箭线图法 ADM` 的可阅读样例文件
5. 更新当前 OpenSpec 规范，使目录组织方式与仓库实际状态一致

## 预期效果

- 学习者打开第09章 `输出物/` 时，能直接看到“过程 -> 输出物样例”的对应关系
- 第10章中 `PDM`、`ADM` 不再只是术语，而有可直接阅读的图示样例
- 章节资产从“知道名词”进一步提升到“看清结构和长相”
