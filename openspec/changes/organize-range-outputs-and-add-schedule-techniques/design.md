# 设计：范围管理输出物按过程归档，进度管理新增工具与技术样例

## 设计目标

让学习者在查看第09章和第10章时，不只知道文件存在，还能迅速对应到过程位置和工具形态。

## 一、第09章输出物目录调整

第09章 `chapters/09-范围管理/输出物/` 保留根目录 `README.md`，并新增以下过程子目录：

1. `01-规划范围管理/`
2. `02-收集需求/`
3. `03-定义范围/`
4. `04-创建WBS/`
5. `05-确认范围/`
6. `06-控制范围/`

文件迁移规则：

- `范围管理计划.md`、`需求管理计划.md` -> `01-规划范围管理/`
- `需求文件.md`、`需求跟踪矩阵.md` -> `02-收集需求/`
- `项目范围说明书.md` -> `03-定义范围/`
- `WBS.md`、`WBS词典.md` -> `04-创建WBS/`
- `验收确认单.md` -> `05-确认范围/`
- `变更请求单.md` -> `06-控制范围/`

## 二、第09章索引方式

根目录 `README.md` 继续保留，但内容调整为：

- 先按过程列出正式输出和当前提供的样例文件
- 再提供“建议连着看”的学习路线

这样既保留总入口，又不会让文件平铺在一起难以定位。

## 三、第10章工具与技术目录

在 `chapters/10-进度管理/` 下新增：

- `工具与技术/README.md`
- `工具与技术/紧前关系绘图法-PDM.md`
- `工具与技术/箭线图法-ADM.md`

内容要求：

- 必须解释图示怎么看
- 必须给出最小可读样例
- 必须指出与考试高频点相关的理解抓手

## 四、章节入口

第10章 `README.md` 新增：

- `工具与技术：[工具与技术/README.md](工具与技术/README.md)`

必要时在 `完整整理.md` 中补充跳转说明，帮助学习者从术语直接跳到样例。

## 五、规范更新

当前规范需要新增两类要求：

1. 第09章 `输出物/` 可以并且应按过程组织样例
2. 第10章可以并且应提供 `工具与技术/` 目录，用于存放高频方法的直观样例
