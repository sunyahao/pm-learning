# 设计：重点提问文档

## 目标

让第08章至第17章除了讲解型和整理型文档之外，再具备一份可直接拿来提问、自测和带学的提问型文档。

## 设计原则

### 1. 仍然保持单章来源边界

每份 `提问文档.md` 只围绕对应 PDF 章节展开，不未标注地合并多个章节内容。

### 2. 问题要服务理解，而不是堆题

文档不做成题库，也不追求覆盖所有零碎知识点，而是优先提问：

- 核心定义
- 过程顺序
- 关键输出物
- 高频对比
- 易错判断
- 案例场景追问

### 3. 问法要适合口头复盘

每份文档按老师带问的节奏组织，适合：

- 学习者自己遮住原文口头回答
- 两人结对互问
- 章节学完后的快速抽查

### 4. 与现有章节资产形成互补

- `精讲.md` 负责讲透主线
- `完整整理.md` 负责补足知识密度
- `提问文档.md` 负责把重点内容转成主动回忆问题

## 文件结构

每份 `提问文档.md` 采用统一结构：

1. 来源说明
2. 使用方式
3. 核心快问
4. 过程串联提问
5. 对比辨析提问
6. 案例追问
7. 临考抽问

## 章节入口

每个重点章节 `README.md` 的“学习入口”增加 `提问文档.md` 链接，使其成为正式学习资产的一部分。
