# 提案：补充章节串联讲解写法

## 为什么要改

当前仓库的章节资产已经能覆盖“章节对齐”“核心考点”“快速复习”等基础目标，但整体写法仍然偏提纲式。

用户新增了一个更具体的要求：希望输出不是简单复述 PDF，而是在不脱离来源章节的前提下，补充学习顺序、概念衔接、跨章连接和考试理解提示，让阅读者更容易把内容真正串起来。

这类加工如果不写入规范，后续产出会出现两个问题：

1. 有的章节只剩零散要点，读者知道“背什么”，但不知道“为什么这样学”。
2. 有的解释性内容虽然有帮助，但没有清楚约束，容易逐步滑向跨章混写或脱离来源。

## 本次变更内容

1. 明确章节学习资产可以在单章来源基础上补充理解型组织加工。
2. 明确这类加工不得破坏 PDF 的章节边界和来源映射。
3. 明确 `guides/` 可以承载跨章节的串联导学与学习地图。
4. 新增一份跨章节串联导学文件。
5. 为第08章至第17章补充 `精讲.md`，形成教师版完整讲解。
6. 将第06章改写为更强解释性的示范版本。
