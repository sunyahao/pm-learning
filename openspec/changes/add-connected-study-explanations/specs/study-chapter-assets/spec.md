## 新增要求

### 要求：章节学习文件可以补充理解型组织加工
章节目录下的 `README.md`、`主脉络.md` 和 `核心记忆.md` 可以在本章来源基础上补充“为什么先学这个”“这一章承接哪章、会支撑哪章”“这类题常在哪一步出错”等理解型组织加工，但这些补充必须服务于理解和复习，不得替代来源映射。

#### 场景：新手阅读某章学习文件
- **前提** 新手第一次系统学习某章
- **当** 他阅读 `README.md`、`主脉络.md` 或 `核心记忆.md`
- **则** 不仅能看到考点提炼
- **并且** 能理解本章的学习顺序、跨章位置和常见混淆点

### 要求：重点项目管理章节必须提供精讲文件
第08章至第17章的章节目录下必须提供 `精讲.md`，用一份完整的 Markdown 文件把本章核心概念、过程链路、关键输出物、跨章连接和常见易错点串联起来。

#### 场景：学习十大知识领域重点章节
- **前提** 读者进入第08章至第17章任一章节目录
- **当** 他查看目录内容并开始系统学习
- **则** 能找到 `精讲.md`
- **并且** 能通过该文件获得比提纲更完整的章节讲解

### 要求：重点项目管理章节必须提供完整整理文件
第08章至第17章的章节目录下必须提供 `完整整理.md`，用于按主来源 PDF 章节顺序整理更完整的知识点内容，并通过 Markdown 结构化呈现概念、过程、关键文档和高频提醒。

#### 场景：读者希望获得接近原 PDF 信息密度的章节内容
- **前提** 读者觉得提纲和精讲仍不足以覆盖重点章节细节
- **当** 他进入第08章至第17章任一章节目录
- **则** 能找到 `完整整理.md`
- **并且** 能阅读到经过整理加工的、接近主来源信息密度的章节内容
