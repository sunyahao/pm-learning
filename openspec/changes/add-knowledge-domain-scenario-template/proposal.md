# 提案：为重点项目管理章节新增过程场景讲解学习资产

## 为什么要改

当前第08章至第17章已经具备：

- `完整整理.md`
- `精讲.md`
- `复盘题单.md`
- `参考作答.md`
- `输出物/`

这些资产已经能覆盖概念、过程、输出物和主动回忆，但仍有一个明显短板：

学习者知道“这个知识领域有哪些术语”，却不一定真的看清“这章里的几个过程和重点概念在项目里到底怎么用”。

尤其对时间紧、以通过考试为目标的学习者来说，纯概念式讲解容易出现三个问题：

1. 过程名字会背，但不知道在日常项目推进中具体在哪一步发生
2. 知道输出物名字，但不知道这些输出物为什么在这个时点出现、由谁来写、拿给谁用
3. 关键概念只会定义，不会放回“真实业务怎么推进”的链条里理解

因此，需要在 OpenSpec 中定义一套统一的“过程场景讲解模板”，并把它正式落到第08章至第19章的重点项目管理章节中。

## 要改什么

1. 在 OpenSpec change 中定义重点项目管理章节的 `过程场景讲解.md` 模板
2. 明确这类文件的目标不是重复概念，而是用一个项目例子解释“这一章里的过程和重点知识在业务里怎么落地”
3. 将该模板正式应用到第08章至第19章
4. 同步更新 README 学习入口和章节导航

## 预期效果

- 学习者能通过同一类项目场景，看清各重点章节中的过程和重点知识点到底怎么用
- 第08章至第19章形成“概念讲解 + 过程落地 + 场景理解”的学习链
- 案例题和论文写作会更容易把抽象术语映射回真实项目动作
