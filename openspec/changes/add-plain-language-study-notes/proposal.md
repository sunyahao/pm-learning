# 提案：为复杂术语补充白话拆解文档

## 为什么要改

当前仓库已经有 `完整整理.md`、`精讲.md`、`过程场景讲解.md` 等学习资产，适合建立体系和过程主线。但对一部分学习者来说，真正卡住的并不是“没看到概念”，而是：

1. 术语字面看得懂，放到公司项目里却不知道到底在干什么
2. 像 `FS/SS/FF/SF`、`PDM/ADM`、`提前量/滞后量` 这类高频概念，容易在定义层面打转，形成“盲人摸象”
3. 阅读 `完整整理.md` 和 `精讲.md` 后，仍然需要一份更口语化、更贴近日常业务的补充讲解

因此，需要新增一种专门服务“术语去抽象化”的学习资产，让复杂概念先变得好懂，再回到标准表述和考试表述。

## 要改什么

1. 在 OpenSpec 中新增 `白话拆解.md` 这一类学习资产定义
2. 明确其用途是：用大白话、业务例子、易混对比解释复杂术语和业务概念
3. 规定如果章节提供 `白话拆解.md`，应放在 `过程场景讲解.md` 之后、`复盘题单.md` 之前
4. 先在第10章 `进度管理` 落一份样板，覆盖 `FS/SS/FF/SF`、`PDM/ADM`、`提前量/滞后量`、`关键路径` 等高频难点

## 预期效果

- 学习者不再只停留在“会背定义”，而是能把术语和真实项目动作对应起来
- 第10章中最抽象的一批进度逻辑概念，有一份专门适合快速理解的入口
- 后续如果其它重点章节也出现类似问题，可以复用同一资产类型继续扩展
