# 提案：在白话拆解中补充“为什么是这些 ITTO”

## 为什么要改

当前重点项目管理章节已经有：

- `完整整理.md`
- `精讲.md`
- `过程场景讲解.md`
- `白话拆解.md`
- `ITTO输出速看.md`

但对很多学习者来说，真正的卡点不是“没看到 ITTO 名称”，而是：

1. 记住了输入、工具和输出名字，却不知道为什么这个过程会吃进这些输入、用这些工具、产出这些输出
2. 容易把 ITTO 当成死表背诵，而不是按管理逻辑去理解
3. 一旦换成案例题或论文语境，就不会把 ITTO 和项目动作对应起来

因此，需要在 `白话拆解.md` 中增加一类解释：不是只说“是什么”，而是讲“为什么这一步天然会有这些 ITTO”，既包括关键输入，也包括关键工具与技术、关键输出。

## 要改什么

1. 在 OpenSpec 中为 `白话拆解.md` 增加 ITTO 成因解释的说明
2. 先在第12章 `质量管理` 落一段样板内容
3. 样板重点围绕质量管理三过程，解释为什么会吃进、使用和产出这些关键 ITTO：
   - 质量管理计划、范围基准、可交付成果、工作绩效数据
   - 标杆对照、成本效益分析、质量成本、审计、根本原因分析、检查、统计抽样、控制图等关键工具
   - 质量管理计划、质量测量指标
   - 质量报告、测试与评估文件、变更请求
   - 质量控制测量结果、核实的可交付成果、工作绩效信息、变更请求

## 预期效果

- 学习者不再只是机械背 ITTO
- 能把 ITTO 和过程目标、项目动作、案例表达联系起来
- 为后续把这套讲法扩到其他知识领域打样
