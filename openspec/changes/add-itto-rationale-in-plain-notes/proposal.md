# 提案：在白话拆解中补充“为什么是这些 ITTO”

## 为什么要改

当前重点项目管理章节已经有：

- `完整整理.md`
- `精讲.md`
- `过程场景讲解.md`
- `白话拆解.md`
- `ITTO输出速看.md`

但对很多学习者来说，真正的卡点不是“没看到 ITTO 名称”，而是：

1. 记住了输入、工具和输出名字，却不知道为什么这个过程会吃进这些输入、用这些工具、产出这些输出
2. 容易把 ITTO 当成死表背诵，而不是按管理逻辑去理解
3. 一旦换成案例题或论文语境，就不会把 ITTO 和项目动作对应起来

因此，需要在 `白话拆解.md` 中增加一类解释：不是只说“是什么”，而是讲“为什么这一步天然会有这些 ITTO”，既包括关键输入，也包括关键工具与技术、关键输出。

## 要改什么

1. 在 OpenSpec 中把“白话拆解解释 ITTO 成因”明确成第08章至第17章统一适用的能力
2. 以第12章 `质量管理` 为样板，把同样的讲法扩展到其余十大知识领域章节
3. 在第08章至第17章的 `ITTO输出速看.md` 中补充回链提示，帮助学习者从“速看”回到“为什么会这样配”
4. 解释重点围绕各章关键过程，说明为什么会吃进关键输入、使用关键工具与技术、产出关键输出

## 预期效果

- 学习者不再只是机械背 ITTO
- 能把 ITTO 和过程目标、项目动作、案例表达联系起来
- 让第08章至第17章形成统一的 ITTO 理解路径，而不是只有第12章有样板
