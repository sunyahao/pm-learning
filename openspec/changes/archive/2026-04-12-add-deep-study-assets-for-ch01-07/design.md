# 设计：第01章至第07章强化学习链

## 设计目标

为第01章至第07章补齐与第08章至第19章风格一致的学习链，但只复用“讲解、场景、白话、复盘”这一层，不引入不适合基础章节的 `ITTO输出速看.md` 和 `输出物/` 结构。

## 章节定位

第01章至第07章属于“基础导入章节”：

- 第01章至第05章负责交代信息化、技术、治理、系统管理与工程背景
- 第06章负责把项目管理的对象、层级、环境和组织概念搭起来
- 第07章负责把项目从“值得做”推进到“可以正式启动”的立项逻辑讲清楚

因此，这些章节最需要的是：

1. 一份接近 PDF 密度的 `完整整理.md`
2. 一份更适合连续阅读的 `精讲.md`
3. 一份把概念放回业务场景的 `过程场景讲解.md`
4. 一份去抽象化的 `白话拆解.md`
5. 一组围绕主动回忆的 `记忆与复盘/` 材料

## 不引入的结构

本次不为第01章至第07章新增以下要求：

- `ITTO输出速看.md`
- `输出物/`

原因是这些章节大多不是以 PMBOK 过程和输出物为主的知识结构，强行套用只会削弱学习效率。

## README 学习入口顺序

第01章至第07章的学习入口建议固定为：

1. `完整整理.md`
2. `精讲.md`
3. `过程场景讲解.md`
4. `白话拆解.md`
5. `复盘题单.md`
6. `参考作答.md`
7. `主脉络.md`
8. `核心记忆.md`
9. `记忆与复盘/README.md`

这样可以保持“先系统讲，再用场景讲，再用白话讲，最后做主动回忆”的节奏。

## 内容写作原则

- 仍以 `高项-三色笔记.pdf` 对应章节为主来源
- 允许补充学习顺序、跨章连接、业务化例子和考试抓手
- 不改写 PDF 章节边界
- 不引入与本章主题无关的扩展事实
- 优先服务时间受限、以通过考试为目标的学习者
