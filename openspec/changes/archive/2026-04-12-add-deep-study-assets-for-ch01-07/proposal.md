# 提案：为第01章至第07章补齐强化学习资产

## 为什么要改

当前仓库中，第08章至第19章已经形成了相对完整的学习链：`完整整理.md`、`精讲.md`、`过程场景讲解.md`、`白话拆解.md`、`记忆与复盘/复盘题单.md`、`记忆与复盘/参考作答.md` 等材料可以让学习者按照“先系统理解，再场景落地，再主动回忆”的顺序复习。

但第01章至第07章目前大多只有 `README.md`、`主脉络.md` 和 `核心记忆.md`，存在几个问题：

1. 作为后续知识域的前置章节，前七章缺少可以系统展开阅读的正式讲解文件
2. 学习者很难把“信息化背景、技术基础、治理、系统管理、系统工程、项目管理概论、立项管理”串成一条完整进入项目管理主线的路径
3. 这些章节虽然不适合硬套 `ITTO输出速看.md` 或 `输出物/`，但同样需要 `完整整理 -> 精讲 -> 场景化理解 -> 复盘题单 -> 参考作答` 这类学习链

因此，需要把第01章至第07章也补齐为更完整的学习资产，但保留它们作为“基础导入章节”的特点，不强行套用过程型章节的产物目录。

## 要改什么

1. 在 OpenSpec 中新增第01章至第07章的强化学习资产要求
2. 明确这些章节至少提供：
   - `完整整理.md`
   - `精讲.md`
   - `过程场景讲解.md`
   - `白话拆解.md`
   - `记忆与复盘/README.md`
   - `记忆与复盘/复盘题单.md`
   - `记忆与复盘/参考作答.md`
3. 明确这些章节的 `README.md` 学习入口顺序
4. 在仓库中实际补齐第01章至第07章上述文档，并补上缺失的 `案例/README.md`

## 预期效果

- 前七章不再只是碎片化导读，而能成为进入十大知识领域之前的完整铺垫
- 学习者能更顺地把“背景知识”和“项目管理主线”连接起来
- 仓库中的章节体验更一致，但仍保留基础章节与过程章节的边界差异
