# 提案：补齐第18章项目绩效域的强化学习资产

## 为什么要改

当前第18章 `项目绩效域` 只有基础骨架：

- `README.md`
- `主脉络.md`
- `核心记忆.md`
- `案例/`

但它缺少重点章节应有的完整学习链：

- `完整整理.md`
- `精讲.md`
- `复盘题单.md`
- `参考作答.md`

这会带来两个问题：

1. 第18章是新版教材的重要内容，也是近年论文与案例已出现的命题方向，但仓库里还没有形成系统学习入口
2. 学习者容易把绩效域只背成“八个名字”，却无法把它和传统知识域、论文表达、案例判断联系起来

## 要改什么

1. 在 `study-chapter-assets` 中新增第18章强化学习资产要求
2. 明确第18章 `README.md` 的学习入口顺序
3. 为第18章补齐 `完整整理.md`、`精讲.md`、`复盘题单.md`、`参考作答.md`
4. 更新章节导航和资料总览，让第18章进入重点强化层

## 预期效果

- 第18章从“可速记”升级为“可系统学习、可口述表达、可应对论文与案例”的重点章节
- 学习者能把八大绩效域和传统知识域、题型场景建立对应关系
- 仓库对新版教材重点的覆盖更加完整，更贴近拿证目标
