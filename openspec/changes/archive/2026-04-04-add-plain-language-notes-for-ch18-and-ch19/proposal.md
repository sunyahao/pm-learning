# 提案：为第18章和第19章补齐白话拆解文档

## 为什么要改

当前仓库里：

- 第18章 `项目绩效域` 已有 `完整整理.md`、`精讲.md`、`过程场景讲解.md`、`复盘题单.md`、`参考作答.md`
- 第19章 `配置与变更管理` 也已形成重点章节学习链

但这两章都缺少 `白话拆解.md`。

这会带来两个问题：

1. 第18章是新版教材视角，最容易被学成“八个名字背一下”，但学习者往往卡在“不知道它到底是在看什么”
2. 第19章概念很多，配置项、基线、配置库、变更闭环、发布回退容易混在一起，仅靠正式整理不够接地气

仓库当前已经要求第08章至第17章提供 `白话拆解.md`。第18章和第19章虽然不在原规则中，但它们同样属于高频、抽象、容易发虚的重点章节，也需要同类学习资产来补上理解层。

## 要改什么

1. 在 `study-chapter-assets` 中新增第18章和第19章的 `白话拆解.md` 要求
2. 明确这两章的 `README.md` 学习入口中，`白话拆解.md` 也应位于 `过程场景讲解.md` 之后、`复盘题单.md` 之前
3. 为第18章补充围绕绩效域视角、八大绩效域、价值与度量的白话拆解
4. 为第19章补充围绕配置项、基线、配置库、变更闭环、发布回退的白话拆解

## 预期效果

- 第18章不再只剩“新版术语感”，而是能让学习者先看懂“绩效域到底是在看什么”
- 第19章不再只剩“流程和审批感”，而是能让学习者看懂为什么配置管理和变更管理必须一起受控
- 第18章和第19章的学习链与前面重点章节更一致，阅读体验更完整
