### 要求：第20章至第24章必须提供延展学习资产
第20章至第24章的章节目录下必须提供 `完整整理.md`、`精讲.md`、`过程场景讲解.md`、`白话拆解.md`、`记忆与复盘/README.md`、`记忆与复盘/复盘题单.md` 和 `记忆与复盘/参考作答.md`，用于把组织级管理、计算基础、治理管理与法律规范章节也组织成“讲解 + 场景 + 白话 + 复盘”的完整学习链。

#### 场景：学习者想系统补齐第20章至第24章
- **前提** 学习者进入第20章至第24章任一章节目录
- **当** 他准备系统理解本章并做主动回忆
- **则** 能找到 `完整整理.md`
- **并且** 能找到 `精讲.md`
- **并且** 能找到 `过程场景讲解.md`
- **并且** 能找到 `白话拆解.md`
- **并且** 能在 `记忆与复盘/` 下找到 `README.md`
- **并且** 能在 `记忆与复盘/` 下找到 `复盘题单.md`
- **并且** 能在 `记忆与复盘/` 下找到 `参考作答.md`

### 要求：第20章至第24章学习入口必须先讲解后提问
第20章至第24章 `README.md` 的学习入口中，`完整整理.md`、`精讲.md`、`过程场景讲解.md` 和 `白话拆解.md` 必须位于 `记忆与复盘/复盘题单.md` 之前，`记忆与复盘/参考作答.md` 必须位于 `记忆与复盘/复盘题单.md` 之后，以体现“先系统理解，再场景消化，再主动回忆，最后核对表达”的顺序。

- **前提** 读者打开第20章至第24章任一章节目录下的 `README.md`
- **当** 他查看学习入口
- **则** 能先看到 `完整整理.md` 和 `精讲.md`
- **并且** 能在 `复盘题单.md` 之前看到 `过程场景讲解.md` 和 `白话拆解.md`
- **并且** 在 `复盘题单.md` 之后看到 `参考作答.md`
