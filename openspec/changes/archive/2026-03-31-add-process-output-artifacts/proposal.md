# 提案：补充过程输出物样例资产

## 为什么要改

当前仓库已经有 `ITTO输出速看.md`，但它只能告诉读者“有哪些输出物”，还不能帮助新手理解“这些输出物长什么样”。

用户明确提出需要：

1. 按核心过程章节补充输出物样例。
2. 把输出物做成独立文件夹而不是只停留在清单。
3. 像“项目范围说明书”这样的输出物，要能直接看到样例文件。

## 本次变更内容

1. 在第08章至第17章以及第19章目录下新增 `输出物/`。
2. 为每章补充 `输出物/README.md`，说明过程与输出物对应关系。
3. 为主要文件型输出物补充可阅读样例文件。
4. 将这项要求写入 OpenSpec 规范。

## 影响范围

- `openspec/specs/`
- `openspec/project.md`
- `chapters/08-17`
- `chapters/19`
