## 新增要求

### 要求：每个章节文件都要具备固定结构
每个派生章节文件都必须包含元数据、本章定位、核心考点、高频关键词、易错点和复盘提示。

#### 场景：打开任一章节文件
- **前提** 读者打开 `chapters/` 下任意章节文件
- **当** 他查看章节结构
- **则** 能看到用于快速复习的固定栏目

### 要求：章节身份必须精确可识别
每个派生章节文件都必须在元数据中标明 `chapter_id`、`source_pdf_chapter` 和 `source_pdf_pages`，并且章节标题必须与 PDF 章节标题一致。

#### 场景：检查章节身份信息
- **前提** 贡献者打开章节文件的元数据区块
- **当** 他阅读元数据和一级标题
- **则** 能识别章节编号
- **并且** 能识别精确的 PDF 章节标题
- **并且** 能识别该章节对应的页码范围
