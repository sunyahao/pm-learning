# 提案：OpenSpec 结构与 PDF 章节对齐改造

## 背景

仓库此前先使用了自定义学习笔记结构，后又出现了与 PDF 目录不一致的 16 章聚合方案，导致仓库结构和章节映射都不规范。

## 本次变更

1. 在仓库根部补齐标准 OpenSpec 文件和目录。
2. 为仓库结构、来源对齐、章节资产和助手工作流建立长期规范。
3. 将 `chapters/` 改为与 PDF 目录一一对应的 24 个章节文件。
4. 将这次改造归档，便于后续追溯结构决策。

## 影响

- 仓库结构符合 OpenSpec 规范。
- AI 工具可以按 `AGENTS.md` + `openspec/` 的标准方式协作。
- 章节数量、顺序和标题与来源 PDF 保持完全一致。
