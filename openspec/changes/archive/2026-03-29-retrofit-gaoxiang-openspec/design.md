# 设计：OpenSpec 结构与 PDF 章节对齐改造

## 决策摘要

这次改造将“规范源头”和“学习输出”明确分层：

1. `openspec/specs/` 作为规范源头。
2. `chapters/` 继续作为直接面向复习的输出层。
3. 来源 PDF 仍然是内容主来源。
4. 章节文件不再使用自定义聚合方式，而是直接镜像 PDF 目录。

## 关键决策

### 保留 `chapters/` 输出层

仓库仍然需要能直接阅读和复习的章节文件，因此 OpenSpec 负责治理这些资产，而不是用规范文件替代它们。

### 删除旧的顶层自定义规范目录

此前的 `specs/` 和 `templates/` 与 `openspec/specs/` 职责重复，因此移除，以保持结构单一且规范。

### 章节目录直接对齐 PDF

用户要求非常明确：章节数量和章节名都必须与 PDF 一致。因此 `chapters/` 使用 24 个直接映射文件，而不是自定义的聚合专题。
