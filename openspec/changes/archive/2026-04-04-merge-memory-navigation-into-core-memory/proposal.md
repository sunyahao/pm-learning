# 提案：将重点记忆导航并入核心记忆

## 为什么要改

此前为第08章新增了 `重点记忆导航.md`，希望补足“临考前快速定位重点并跳转”的场景。

但落地后发现两个问题：

1. `重点记忆导航.md` 与 `核心记忆.md` 的定位高度重叠，都会被学习者理解为“临考速记稿”。
2. 在当前 IDEA 阅读环境里，跨文件锚点跳转不稳定，独立导航文件并没有稳定带来更好的定位体验，反而增加了维护成本。

因此，需要把“结构化速记 + 快速跳转”的能力并回 `核心记忆.md`，只保留一份统一的临考速记入口。

## 要改什么

1. 取消把 `重点记忆导航.md` 作为独立学习资产的要求。
2. 改为允许并鼓励在 `核心记忆.md` 中内置“快速跳转 + 分块记忆 + 展开阅读链接”结构。
3. 用第08章 `整合管理` 作为首个增强型 `核心记忆.md` 样例。
4. 删除第08章中重复的 `重点记忆导航.md` 和对应入口。

## 预期效果

- 学习者只需要记住一份临考速记稿，不必在两份相似材料之间切换。
- 跳转体验优先依赖“本页精准定位”，减少对跨文件锚点兼容性的依赖。
- 后续第09章至第17章如需要增强速记能力，可直接扩展 `核心记忆.md`，不再新增平行资产类型。
