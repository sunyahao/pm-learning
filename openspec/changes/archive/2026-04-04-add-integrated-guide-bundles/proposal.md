# 提案：新增项目管理统合理解导学目录

## 为什么要改

当前 `guides/` 已经有学习地图、冲刺计划和章节导航，但还缺一类更“统合”的资料：

1. 把十大知识领域、五大过程组和 49 个过程放在一起总览。
2. 把大量跨过程重复出现的 `事业环境因素`、`组织过程资产`、`专家判断`、`数据收集`、`人际关系与团队技能`、`会议` 等共性 ITTO 统一解释。
3. 给已经看过各知识领域、正准备进入“记忆 + 刷题”阶段的学习者一个跨章节的总入口。

用户还明确提出两项新需求：

1. 新建一个用于“十大领域统合理解”的总览目录。
2. 将现有 `guides/十大知识领域与五大过程组交叉矩阵.md` 移入该目录，并在该目录中继续补充共性 ITTO 资料。

## 本次变更内容

1. 在 `guides/` 下新增一个面向项目管理统合理解的子目录。
2. 将“十大知识领域与五大过程组交叉矩阵”移入该子目录。
3. 新增一份解释 `事业环境因素`、`组织过程资产` 和通用工具技术的总览文档。
4. 为该子目录补充 `README.md`，说明适用阶段和阅读顺序。
5. 更新 OpenSpec 规范，明确 `guides/` 可以使用主题化子目录承载跨章节的统合学习资产。

## 影响范围

- `openspec/specs/study-repository/spec.md`
- `openspec/changes/2026-04-04-add-integrated-guide-bundles/`
- `guides/`
