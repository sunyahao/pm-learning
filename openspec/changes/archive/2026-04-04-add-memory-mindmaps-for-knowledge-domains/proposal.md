# 提案：为重点知识领域补充记忆型思维导图资产

## 为什么要改

当前重点项目管理章节已经有：

- `完整整理.md`
- `精讲.md`
- `过程场景讲解.md`
- `白话拆解.md`
- `复盘题单.md`
- `参考作答.md`

这些资产适合系统学习和主动回忆，但对“临考前快速收束本章记忆主线”这一场景，还缺一类更压缩、更图形化的材料。

尤其是十大知识领域，学习者常常会遇到这类问题：

1. 单个知识点会背，但脑子里没有总图，不知道它和过程、输出物、易混点怎么连起来
2. 章节看完后信息很多，缺一份能在几分钟内整体扫过的记忆总览
3. 临考时更需要“主线 + 过程 + 易混点 + 答题抓手”的树状结构，而不是再看一遍长文

因此，需要新增一种“重点记忆导航”学习资产，用于把章节最值得记忆的内容收束成结构化清单，并直接链接到对应知识点和样例文件。

## 要改什么

1. 在 `study-chapter-assets` 中定义 `重点记忆导航.md` 这一类学习资产
2. 明确其用途是：围绕章节主线、过程、关键输出物、易混点和答题抓手，形成临考速览
3. 明确如果章节提供 `重点记忆导航.md`，应放在 `核心记忆.md` 之后，作为速记补充
4. 先在第08章 `整合管理` 落一份示例，作为十大知识领域后续复用的模板

## 预期效果

- 学习者能在正式学习材料之外，多一份适合快速扫重点、并且可以直接跳到知识点的总览材料
- 第08章可以先形成“文本讲解 + 记忆图谱”的双层入口
- 后续第09章至第17章如继续扩展，也能沿用同一资产类型和组织方式
