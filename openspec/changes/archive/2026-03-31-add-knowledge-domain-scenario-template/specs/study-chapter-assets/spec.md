## ADDED Requirements

### 要求：重点项目管理章节的过程场景讲解必须围绕单一项目例子展开
第08章至第19章如新增 `过程场景讲解.md`，必须围绕一个可复用的单一项目场景展开，用同一项目背景解释本章过程、关键知识点、文档和上下游交接关系，而不是把多个零散小例子堆叠在一起。

#### 场景：学习者想用一个项目例子看清过程怎么在业务里发生
- **前提** 学习者进入第08章至第19章任一章节目录并打开 `过程场景讲解.md`
- **当** 他顺着文档从头阅读
- **则** 能看到明确的项目背景
- **并且** 能在同一项目语境下理解本章过程、关键知识点和文档
- **并且** 不会因为混入多个项目例子而失去主线

### 要求：重点项目管理章节的过程场景讲解必须解释“过程和知识点怎么落地”
第08章至第19章如新增 `过程场景讲解.md`，文档必须至少包含以下部分：`场景底稿`、`这一章在项目里到底负责什么`、`不做或做差会发生什么`、`这一章的典型动作链`、`这一章常见文档在场景里长什么样`、`日常业务里最容易卡住的点`、`这一章和其他知识领域怎么接力`、`案例题会怎么考`、`论文里怎么写更像做过项目`、`一句话记忆`。

#### 场景：学习者觉得知识点只有定义没有业务动作
- **前提** 学习者已经看过某章概念整理，但仍觉得抽象
- **当** 他阅读该章 `过程场景讲解.md`
- **则** 能看清本章在项目中的责任边界
- **并且** 能看清过程链、关键知识点和文档用途
- **并且** 能把本章内容映射到案例题和论文表达

### 要求：重点项目管理章节的过程场景讲解必须突出边界、后果和业务卡点
第08章至第19章如新增 `过程场景讲解.md`，必须明确写出“本知识领域不直接解决什么”“这一章没做好会导致什么项目后果”以及“日常业务里为什么经常卡在这里”，避免再次把重点项目管理章节写成互相重叠的概念清单。

#### 场景：学习者总把多个知识领域混在一起
- **前提** 学习者在做案例题时分不清应该优先调用哪个知识领域
- **当** 他阅读 `过程场景讲解.md` 中的边界、后果和业务卡点部分
- **则** 能更快判断本章的主战场
- **并且** 能知道该问题应联动哪些其他章节
