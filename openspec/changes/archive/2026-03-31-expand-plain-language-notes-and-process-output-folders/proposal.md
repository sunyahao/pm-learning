# 提案：扩展知识领域白话拆解，并将核心章节输出物按过程归档

## 为什么要改

当前仓库已经在第10章 `进度管理` 试点了 `白话拆解.md`，也在第09章 `范围管理` 试点了“输出物按过程目录归档”的结构。这两个方向已经证明有明显价值：

1. 对抽象术语和复杂逻辑，单靠正式定义和精讲仍然不够，学习者需要一层更口语化、更贴近业务动作的解释
2. 对输出物样例，单纯把文件平铺在 `输出物/` 根目录下，学习者仍然不容易看清“这个文件属于哪个过程”

但目前这两个能力都只覆盖了局部章节：

- `白话拆解.md` 只有第10章有
- 输出物按过程归档只有第09章有

这会让仓库出现强弱不均的问题。为了更好服务高项备考，需要把这两种设计扩展到整个项目管理核心章节带。

## 要改什么

1. 为第08章至第17章统一补充 `白话拆解.md`
2. 将第08章至第17章以及第19章的 `输出物/` 全部改为按过程建立子目录
3. 保留每章 `输出物/README.md` 作为总索引，但索引要显式按过程链接到对应文件
4. 更新章节 `README.md`、导航文件和仓库内相关引用，确保学习入口和文件路径一致
5. 更新 OpenSpec 当前规范，使规则与仓库实际状态一致

## 预期效果

- 十大知识领域都拥有“正式讲解 + 场景讲解 + 白话拆解”的完整理解链
- 核心章节输出物不再只是“知道名字”，而是能一眼看清属于哪个过程
- 学习者更容易把“概念、过程、样例文件、答题表达”串成一套
