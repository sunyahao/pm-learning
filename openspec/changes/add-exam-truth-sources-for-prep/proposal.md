# 提案：把历年真题明确纳入 OpenSpec 事实来源并补资料盘点导航

## 为什么要改

当前 `openspec/project.md` 已经把 `2015-2024年高级历年真题合集/` 视为辅助来源，但规范层还不够具体，主要有三个问题：

1. 没有明确区分论文真题、案例真题和选择题真题各自应该支撑什么派生资产
2. 没有明确要求真题类材料在使用时记录年份、批次和答案依据
3. 仓库虽然已经有章节学习链和冲刺计划，但还缺少一份面向“通过考试”的资料总览与优化建议，帮助读者判断先看什么最提分

这会导致一个实际问题：真题资料虽然在仓库里，但在 OpenSpec 层面还没有真正成为“正式事实来源”。

## 要改什么

1. 在 `source-alignment` 中明确：历年论文、案例、选择题真题属于正式题型来源
2. 规定不同派生资产必须匹配对应题型来源
3. 规定真题引用必须标明年份、批次和答案依据
4. 在 `study-repository` 中允许 `guides/` 提供资料总览与提分导航文档
5. 新增一份面向高项通过目标的资料盘点与优化建议文档

## 预期效果

- OpenSpec 不再只知道“有真题目录”，而是知道“真题怎么用”
- 后续论文趋势、案例索引和上午题高频点整理都有明确来源边界
- 读者能更快判断当前仓库里什么最值得看，减少无效翻资料的时间
