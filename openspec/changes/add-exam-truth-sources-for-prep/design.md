# 设计：真题来源分层与提分导航

## 设计目标

把仓库中的历年真题从“存在于目录中的资料”升级为“在 OpenSpec 中有明确用途和边界的正式事实来源”，并补一份直接服务备考决策的资料总览。

## 一、来源分层

### 1. 章节主来源

- `高项-三色笔记.pdf`

用途：

- 决定 `chapters/` 的 24 章映射
- 约束章节标题、顺序和边界
- 作为章节学习资产的主线依据

### 2. 题型真题来源

- `2015-2024论文写作真题/`
- `2015-2024年案例分析历年真题/`
- `2015-2024年选择题真题/`

用途：

- 论文趋势、押题判断、写作素材组织
- 案例题型归类、案例索引、标准作答校准
- 上午题高频点、易错点、计算题训练

### 3. 辅助讲解来源

- 备考指导、高频考点、模拟试题、优秀论文范文、知识汇总与整理等

用途：

- 提炼考试经验
- 强化理解
- 提供写作表达和冲刺建议

## 二、真题引用规则

凡是基于历年真题产出的文档，都应尽量明确以下信息：

- 年份
- 上下半年
- 批次
- 题型
- 题干依据
- 答案依据

如果同时存在 `无答案版` 与 `有答案版`：

- `无答案版` 优先用于还原题干
- `有答案版` 用于答案校准
- 若仓库给出的是自主整理答案，应显式说明

## 三、提分导航文档

新增一份 `guides/资料盘点与提分优化建议.md`，目的不是重复已有冲刺计划，而是解决三个问题：

1. 当前仓库里哪类资料最重要
2. 哪些部分已经够用
3. 哪些地方最影响通过率，应该优先补

这份文档会直接面向“上班族、时间有限、目标是通过考试”的场景，不做纯目录式罗列。
