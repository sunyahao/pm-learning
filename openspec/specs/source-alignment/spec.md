# 来源对齐规范

## 目的

确保所有学习资产严格对齐来源 PDF，并精确镜像 PDF 的目录结构。

## 要求

### 要求：必须明确主来源文档
仓库必须将 `高项-三色笔记.pdf` 标记为当前学习内容集的主来源文档。

#### 场景：查询当前主来源
- **前提** 贡献者需要确认当前内容受哪份文档约束
- **当** 他阅读项目上下文或来源对齐规范
- **则** 能明确看到 `高项-三色笔记.pdf` 被标记为主来源

### 要求：历年真题必须作为正式题型来源
仓库必须将 `2015-2024论文写作真题/`、`2015-2024年案例分析历年真题/` 和 `2015-2024年选择题真题/` 视为正式题型来源，用于支撑论文趋势判断、案例题型归类、上午题高频点提炼和训练材料校准。

#### 场景：维护者需要判断某类题应该依据哪些来源
- **前提** 维护者正在编写论文趋势、案例索引、选择题高频点或冲刺训练材料
- **当** 他查看来源规则
- **则** 能确认历年真题属于正式事实来源
- **并且** 能区分论文、案例和选择题各自对应的真题目录

### 要求：不同派生资产必须匹配对应题型来源
章节学习资产继续以 `高项-三色笔记.pdf` 为主来源；论文趋势、案例索引、上午题训练和押题判断等跨章节材料，必须优先使用与之匹配的历年真题来源，不得只凭讲义或范文替代真题依据。

#### 场景：编写一份论文趋势或案例训练材料
- **前提** 维护者正在编写 `guides/`、章节 `案例/` 或其他训练型文档
- **当** 他需要引用“近几年考了什么”“常见问法是什么”“标准答案怎么看”
- **则** 会优先查对应题型的历年真题
- **并且** 不会仅凭辅助讲解资料直接下结论

### 要求：真题引用必须标明年份批次和答案依据
凡是基于历年真题生成的趋势、索引、训练或讲解材料，都必须至少标明题目的年份、上下半年与批次；若同时存在 `无答案版` 与 `有答案版`，应明确题干依据和答案依据分别来自哪个版本。

#### 场景：维护者整理某年案例题或选择题答案
- **前提** 某道题同时存在 `无答案版` 和 `有答案版`
- **当** 维护者在派生文档中引用题目或答案
- **则** 能明确说明题干取自哪个版本
- **并且** 能明确说明答案校对依据是官方有答案版还是仓库自行整理

### 要求：策略类文件可以使用辅助来源
位于 `guides/` 下的备考策略、趋势分析和冲刺计划可以综合使用备考指导、历年真题和其他备考资料作为辅助来源，但不得改变章节目录与主来源 PDF 的一一对应关系。

#### 场景：编写总览型备考策略
- **前提** 维护者正在编写 `guides/` 下的策略文件
- **当** 他需要判断高频考点、失分点或备考节奏
- **则** 可以引用 `2.备考指导/`、历年真题和其他备考目录中的材料
- **并且** 不会因此修改章节目录对主来源 PDF 的映射规则

### 要求：章节目录必须与 PDF 一一对应
仓库必须在 `chapters/` 下维护 24 个章节目录，并使其数量、顺序和标题与 PDF 目录一一对应。

#### 场景：核对章节目录
- **前提** 读者打开 `chapters/`
- **当** 他将仓库目录与 PDF 目录进行比对
- **则** 章节目录数量与 PDF 的 24 章一致
- **并且** 目录顺序与 PDF 章节顺序一致
- **并且** 目录名称与 PDF 章节标题一致

### 要求：每章都要记录精确来源信息
每个章节目录下的 `README.md` 都必须记录对应的 PDF 章节标题和大致页码范围。

#### 场景：查看章节头部元数据
- **前提** 读者打开任一章节目录下的 `README.md`
- **当** 他查看元数据区块
- **则** 能看到对应的 PDF 章节名
- **并且** 能看到该章对应的 PDF 页码范围

### 要求：理解型加工不得破坏来源边界
派生学习资产可以在单章来源基础上补充学习顺序、跨章连接、考试理解提示等组织加工，但不得改写 PDF 章节边界、不得把多个 PDF 章节当作一个章节内容输出，也不得引入与本章主题无关且无法回溯的事实结论。

#### 场景：检查带有解释性讲解的章节文件
- **前提** 贡献者打开某章的 `README.md`、`主脉络.md` 或 `核心记忆.md`
- **当** 他阅读其中的学习顺序、跨章连接和理解提示
- **则** 能确认这些内容仍然围绕该 PDF 章节展开
- **并且** 不会看到未标注的多章合并改写
