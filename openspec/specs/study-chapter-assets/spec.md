# 章节学习资产规范

### 要求：重点项目管理章节必须提供参考作答文档
第08章至第17章的章节目录下必须提供 `参考作答.md`，用于按 `复盘题单.md` 的问题顺序提供参考答案和作答要点，支持自测后核对、结对复习和带学复盘。
#### 场景：学习者在口头回答后核对答案
- **前提** 学习者已经使用 `复盘题单.md` 做过一轮主动回忆
- **当** 他需要核对自己的回答是否完整
- **则** 能在同章节目录下找到 `参考作答.md`
- **并且** 文档中的答案与问题编号保持对应

### 要求：重点项目管理章节学习入口必须先讲解后提问
第08章至第17章的 `README.md` 学习入口中，`复盘题单.md` 必须位于 `精讲.md` 之后，`参考作答.md` 必须位于 `复盘题单.md` 之后，以体现“先系统讲解，再复习思考，再核对答案”的顺序。
#### 场景：读者按章节入口开始学习重点章节
- **前提** 读者进入第08章至第17章任一章节目录
- **当** 他查看 `README.md` 的学习入口
- **则** 能先看到 `完整整理.md` 和 `精讲.md`
- **并且** 之后看到 `复盘题单.md`
- **并且** 在 `复盘题单.md` 之后看到 `参考作答.md`

## 目的

定义章节学习文件的最小结构、命名要求和内容质量下限。

## 要求

### 要求：每个章节目录都要具备固定结构
每个派生章节目录都必须至少包含 `README.md`、`主脉络.md`、`核心记忆.md` 和 `案例/` 目录。

#### 场景：打开任一章节目录
- **前提** 读者打开 `chapters/` 下任意章节目录
- **当** 他查看目录内容
- **则** 能看到 `README.md`
- **并且** 能看到 `主脉络.md`
- **并且** 能看到 `核心记忆.md`
- **并且** 能看到 `案例/` 目录

### 要求：章节身份必须精确可识别
每个派生章节目录下的 `README.md` 都必须在元数据中标明 `chapter_id`、`source_pdf_chapter` 和 `source_pdf_pages`，并且章节标题必须与 PDF 章节标题一致。

#### 场景：检查章节身份信息
- **前提** 贡献者打开章节文件的元数据区块
- **当** 他阅读元数据和一级标题
- **则** 能识别章节编号
- **并且** 能识别精确的 PDF 章节标题
- **并且** 能识别该章节对应的页码范围

### 要求：主脉络文件必须面向新手
每个章节目录下的 `主脉络.md` 都必须帮助新手快速理解“这章学什么、先学什么、为什么这样学、和哪几章一起看”。

#### 场景：新手第一次打开某章
- **前提** 新手首次进入某章节目录
- **当** 他阅读 `主脉络.md`
- **则** 能快速知道本章主线
- **并且** 能知道学习顺序和重点抓手

### 要求：章节学习文件可以补充理解型组织加工
章节目录下的 `README.md`、`主脉络.md` 和 `核心记忆.md` 可以在本章来源基础上补充“为什么先学这个”“这一章承接哪章、会支撑哪章”“这类题常在哪一步出错”等理解型组织加工，但这些补充必须服务于理解和复习，不得替代来源映射。

#### 场景：新手阅读某章学习文件
- **前提** 新手第一次系统学习某章
- **当** 他阅读 `README.md`、`主脉络.md` 或 `核心记忆.md`
- **则** 不仅能看到考点提炼
- **并且** 能理解本章的学习顺序、跨章位置和常见混淆点

### 要求：重点项目管理章节必须提供精讲文件
第08章至第17章的章节目录下必须提供 `精讲.md`，用一份完整的 Markdown 文件把本章核心概念、过程链路、关键输出物、跨章连接和常见易错点串联起来。

#### 场景：学习十大知识领域重点章节
- **前提** 读者进入第08章至第17章任一章节目录
- **当** 他查看目录内容并开始系统学习
- **则** 能找到 `精讲.md`
- **并且** 能通过该文件获得比提纲更完整的章节讲解

### 要求：重点项目管理章节必须提供完整整理文件
第08章至第17章的章节目录下必须提供 `完整整理.md`，用于按主来源 PDF 章节顺序整理更完整的知识点内容，并通过 Markdown 结构化呈现概念、过程、关键文档和高频提醒。

#### 场景：读者希望获得接近原 PDF 信息密度的章节内容
- **前提** 读者觉得提纲和精讲仍不足以覆盖重点章节细节
- **当** 他进入第08章至第17章任一章节目录
- **则** 能找到 `完整整理.md`
- **并且** 能阅读到经过整理加工的、接近主来源信息密度的章节内容

### 要求：核心记忆文件必须服务快速复习
每个章节目录下的 `核心记忆.md` 都必须提炼该章最值得记忆的高频内容，并说明应优先跳转阅读的文件。

#### 场景：上班族利用碎片时间复习某章
- **前提** 学习者只有很短的复习时间
- **当** 他打开某章节目录下的 `核心记忆.md`
- **则** 能快速看到该章最该背的概念、流程或对比点
- **并且** 能看到继续深挖时应跳转的文件路径

### 要求：不允许无标注聚合多个 PDF 章节
派生章节文件不得在未明确说明的情况下合并多个 PDF 章节。

#### 场景：验证章节边界
- **前提** 贡献者将章节文件与 PDF 目录进行比对
- **当** 他查看文件的来源映射
- **则** 每个文件对应一个 PDF 章节
- **并且** 不存在未标注的多章合并情况

### 要求：章节内容必须面向考试复习
派生章节目录下的 `README.md` 必须提供面向考试的核心考点、关键词和易错点，而不是只保留泛泛概述。

#### 场景：用章节文件进行考前复习
- **前提** 读者在考试前快速回看某章
- **当** 他阅读章节内容
- **则** 能快速识别该章常考内容
- **并且** 能看到关键词和易错点提示

### 要求：学习资产应优先服务时间受限考生
当学习资产设计需要在“覆盖面更广”和“复习效率更高”之间权衡时，应优先支持时间受限且以通过考试为目标的学习者。

#### 场景：在有限时间内安排学习内容
- **前提** 学习者每周可投入时间有限
- **当** 仓库提供备考策略、章节导航和记忆材料
- **则** 这些材料优先突出高频考点、核心流程、关键输出物和易错点

### 要求：项目管理核心章节必须提供 ITTO 输出速看
第08章至第17章以及第19章的章节目录下必须提供 `ITTO输出速看.md`，用于快速复习关键输出物和常见文档。

#### 场景：复习项目管理过程输出物
- **前提** 读者正在复习项目管理核心章节
- **当** 他进入第08章至第17章或第19章目录
- **则** 能找到 `ITTO输出速看.md`
- **并且** 能快速查看该章高频输出物和常见文件

### 要求：每章必须预留案例归档位置
每个章节目录都必须包含 `案例/` 目录，用于放置真题、案例索引和相关材料。

#### 场景：整理章节案例
- **前提** 读者想把某章相关真题或案例材料归入对应章节
- **当** 他进入该章节目录
- **则** 能直接使用 `案例/` 目录进行整理

### 要求：核心项目管理章节必须提供输出物文件夹
第08章至第17章以及第19章的章节目录下必须包含 `输出物/` 目录，用于存放关键输出物样例文件。

#### 场景：查看核心章节的输出物样例
- **前提** 读者正在学习核心项目管理章节
- **当** 他进入第08章至第17章或第19章目录
- **则** 能找到 `输出物/` 目录
- **并且** 目录内包含该章输出物说明或样例文件

### 要求：输出物文件夹必须提供样例
核心项目管理章节的 `输出物/` 目录必须至少包含 `README.md`，并为主要文件型输出物提供可直接阅读的样例文件。

#### 场景：理解“项目范围说明书”长什么样
- **前提** 新手正在学习范围管理
- **当** 他打开第09章目录下的 `输出物/`
- **则** 能找到 `项目范围说明书` 的样例文件
- **并且** 能通过样例理解该输出物的典型结构和内容

### 要求：重点项目管理章节必须提供复盘题单
第08章至第17章的章节目录下必须提供 `复盘题单.md`，用于围绕本章重点概念、过程顺序、关键输出物、高频易混点和案例场景提出结构化问题，服务主动回忆、自测和带学复盘。
#### 场景：学习者在读完重点章节后进行口头自测
- **前提** 学习者已经阅读第08章至第17章中的某一章节学习材料
- **当** 他进入该章节目录，准备做主动回忆或与他人互问
- **则** 能找到 `复盘题单.md`
- **并且** 文档中的问题围绕本章重点内容组织
- **并且** 问题既覆盖记忆性提问，也覆盖理解性和案例性追问
