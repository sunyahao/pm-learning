# 章节学习资产规范

### 要求：核心章节输出物应按过程归档
第08章至第17章以及第19章的 `输出物/` 目录必须按过程或步骤建立子目录，根目录 `README.md` 负责作为总索引，把样例文件明确归到对应过程下，帮助学习者直接理解“哪个文件属于哪一步”。
#### 场景：学习者想用输出物理解过程链
- **前提** 学习者打开核心章节的 `输出物/README.md`
- **当** 他查看输出物索引
- **则** 能按过程看到文件归属
- **并且** 能进入对应过程子目录查看样例文件

### 要求：核心记忆可以内置快速跳转结构
第08章至第19章的 `核心记忆.md` 可以在不重复独立速记文件的前提下，内置“快速跳转 + 分块记忆 + 展开阅读链接”的结构，用于把主线、过程、关键输出物、高频易混点和答题抓手收束到同一份临考速记稿中。

#### 场景：学习者想在短时间内快速扫过一章主线
- **前提** 学习者已经看过正式讲解材料
- **当** 他在考前或碎片时间复习某章
- **则** 能在该章的 `核心记忆.md` 中看到结构化重点列表
- **并且** 该列表能帮助他快速回忆本章重点
- **并且** 关键条目能跳转到本页具体位置或对应样例文件

### 要求：第08章整合管理必须提供增强型核心记忆样例
第08章 `整合管理` 必须在 `核心记忆.md` 中提供一份带快速跳转的样例，用于围绕整合目标、七过程、关键输出与易混点形成一份可复习、可跳转的唯一速记入口。

#### 场景：学习者想先看整合管理速记稿再复习细节
- **前提** 学习者进入 `chapters/08-整合管理/`
- **当** 他查看本章学习材料
- **则** 能在 `核心记忆.md` 中看到结构化速记入口
- **并且** 文档能帮助他快速回忆整合管理主线

### 要求：重点项目管理章节应提供记忆与复盘目录入口
第08章至第19章的章节目录下应提供 `记忆与复盘/README.md`，用于把 `核心记忆.md`、`记忆与复盘/复盘题单.md` 和 `记忆与复盘/参考作答.md` 汇总为统一入口，帮助学习者在临考速记、自测提问和核对表达之间快速切换。

#### 场景：学习者只想集中使用速记与复盘材料
- **前提** 学习者进入第08章至第19章任一章节目录
- **当** 他只想查看临考速记、自测问题和参考答案
- **则** 能找到 `记忆与复盘/README.md`
- **并且** 能从该目录入口快速进入 `核心记忆.md`、`记忆与复盘/复盘题单.md` 和 `记忆与复盘/参考作答.md`

### 要求：重点项目管理章节学习入口应提供记忆与复盘目录
第08章至第19章的 `README.md` 学习入口中应提供 `记忆与复盘/README.md` 的入口，用于明确“速记、自测、核对答案”是一组可集中访问的学习资产。

#### 场景：读者从章节入口寻找速记和复盘材料
- **前提** 读者打开第08章至第19章任一章节目录下的 `README.md`
- **当** 他查看学习入口
- **则** 能看到 `记忆与复盘/README.md` 的入口

### 要求：第10章进度管理应提供工具与技术样例目录
第10章 `进度管理` 可以提供 `工具与技术/` 目录，用于存放高频工具与技术的直观样例。对 `紧前关系绘图法 PDM` 和 `箭线图法 ADM` 这类仅靠术语难以理解的内容，应提供单独的示例文件，并在章节入口中给出跳转。
#### 场景：学习者看不懂 PDM 和 ADM 的图示长相
- **前提** 学习者进入 `chapters/10-进度管理/`
- **当** 他查看章节入口或 `工具与技术/`

### 要求：第11章成本管理必须提供估算方法工具与技术说明
第11章 `成本管理` 必须提供 `工具与技术/` 目录，并提供专门说明 `类比估算`、`参数估算`、`自下而上估算`、`三点估算` 的白话文档，帮助学习者区分这些成本估算方法的使用方式。

#### 场景：学习者分不清几种成本估算方法
- **前提** 学习者正在复习第11章 `估算成本`

第08章至第19章的章节目录下必须在 `记忆与复盘/` 中提供 `参考作答.md`，用于按 `复盘题单.md` 的问题顺序提供参考答案和作答要点，支持自测后核对、结对复习和带学复盘。
#### 场景：学习者在口头回答后核对答案
- **前提** 学习者已经使用 `复盘题单.md` 做过一轮主动回忆
- **当** 他需要核对自己的回答是否完整
- **则** 能在同章节目录的 `记忆与复盘/` 下找到 `参考作答.md`
- **并且** 文档中的答案与问题编号保持对应

### 要求：重点项目管理章节学习入口必须先讲解后提问
第08章至第17章的 `README.md` 学习入口中，`复盘题单.md` 必须位于 `精讲.md` 之后，`参考作答.md` 必须位于 `复盘题单.md` 之后，以体现“先系统讲解，再复习思考，再核对答案”的顺序。
#### 场景：读者按章节入口开始学习重点章节
- **前提** 读者进入第08章至第17章任一章节目录
- **当** 他查看 `README.md` 的学习入口
- **则** 能先看到 `完整整理.md` 和 `精讲.md`
- **并且** 之后看到 `复盘题单.md`
- **并且** 在 `复盘题单.md` 之后看到 `参考作答.md`

## 目的

定义章节学习文件的最小结构、命名要求和内容质量下限。

## 要求

### 要求：每个章节目录都要具备固定结构
每个派生章节目录都必须至少包含 `README.md`、`主脉络.md`、`核心记忆.md` 和 `案例/` 目录。

#### 场景：打开任一章节目录
- **前提** 读者打开 `chapters/` 下任意章节目录
- **当** 他查看目录内容
- **则** 能看到 `README.md`
- **并且** 能看到 `主脉络.md`
- **并且** 能看到 `核心记忆.md`
- **并且** 能看到 `案例/` 目录

### 要求：章节身份必须精确可识别
每个派生章节目录下的 `README.md` 都必须在元数据中标明 `chapter_id`、`source_pdf_chapter` 和 `source_pdf_pages`，并且章节标题必须与 PDF 章节标题一致。

#### 场景：检查章节身份信息
- **前提** 贡献者打开章节文件的元数据区块
- **当** 他阅读元数据和一级标题
- **则** 能识别章节编号
- **并且** 能识别精确的 PDF 章节标题
- **并且** 能识别该章节对应的页码范围

### 要求：主脉络文件必须面向新手
每个章节目录下的 `主脉络.md` 都必须帮助新手快速理解“这章学什么、先学什么、为什么这样学、和哪几章一起看”。

#### 场景：新手第一次打开某章
- **前提** 新手首次进入某章节目录
- **当** 他阅读 `主脉络.md`
- **则** 能快速知道本章主线
- **并且** 能知道学习顺序和重点抓手

### 要求：章节学习文件可以补充理解型组织加工
章节目录下的 `README.md`、`主脉络.md` 和 `核心记忆.md` 可以在本章来源基础上补充“为什么先学这个”“这一章承接哪章、会支撑哪章”“这类题常在哪一步出错”等理解型组织加工，但这些补充必须服务于理解和复习，不得替代来源映射。

#### 场景：新手阅读某章学习文件
- **前提** 新手第一次系统学习某章
- **当** 他阅读 `README.md`、`主脉络.md` 或 `核心记忆.md`
- **则** 不仅能看到考点提炼
- **并且** 能理解本章的学习顺序、跨章位置和常见混淆点

### 要求：重点项目管理章节必须提供精讲文件
第08章至第17章的章节目录下必须提供 `精讲.md`，用一份完整的 Markdown 文件把本章核心概念、过程链路、关键输出物、跨章连接和常见易错点串联起来。

#### 场景：学习十大知识领域重点章节
- **前提** 读者进入第08章至第17章任一章节目录
- **当** 他查看目录内容并开始系统学习
- **则** 能找到 `精讲.md`
- **并且** 能通过该文件获得比提纲更完整的章节讲解

### 要求：重点项目管理章节必须提供完整整理文件
第08章至第17章的章节目录下必须提供 `完整整理.md`，用于按主来源 PDF 章节顺序整理更完整的知识点内容，并通过 Markdown 结构化呈现概念、过程、关键文档和高频提醒。

#### 场景：读者希望获得接近原 PDF 信息密度的章节内容
- **前提** 读者觉得提纲和精讲仍不足以覆盖重点章节细节
- **当** 他进入第08章至第17章任一章节目录
- **则** 能找到 `完整整理.md`
- **并且** 能阅读到经过整理加工的、接近主来源信息密度的章节内容

### 要求：核心记忆文件必须服务快速复习
每个章节目录下的 `核心记忆.md` 都必须提炼该章最值得记忆的高频内容，并说明应优先跳转阅读的文件。

#### 场景：上班族利用碎片时间复习某章
- **前提** 学习者只有很短的复习时间
- **当** 他打开某章节目录下的 `核心记忆.md`
- **则** 能快速看到该章最该背的概念、流程或对比点
- **并且** 能看到继续深挖时应跳转的文件路径

### 要求：不允许无标注聚合多个 PDF 章节
派生章节文件不得在未明确说明的情况下合并多个 PDF 章节。

#### 场景：验证章节边界
- **前提** 贡献者将章节文件与 PDF 目录进行比对
- **当** 他查看文件的来源映射
- **则** 每个文件对应一个 PDF 章节
- **并且** 不存在未标注的多章合并情况

### 要求：章节内容必须面向考试复习
派生章节目录下的 `README.md` 必须提供面向考试的核心考点、关键词和易错点，而不是只保留泛泛概述。

#### 场景：用章节文件进行考前复习
- **前提** 读者在考试前快速回看某章
- **当** 他阅读章节内容
- **则** 能快速识别该章常考内容
- **并且** 能看到关键词和易错点提示

### 要求：学习资产应优先服务时间受限考生
当学习资产设计需要在“覆盖面更广”和“复习效率更高”之间权衡时，应优先支持时间受限且以通过考试为目标的学习者。

#### 场景：在有限时间内安排学习内容
- **前提** 学习者每周可投入时间有限
- **当** 仓库提供备考策略、章节导航和记忆材料
- **则** 这些材料优先突出高频考点、核心流程、关键输出物和易错点

第08章至第17章以及第19章的章节目录下必须提供 `ITTO输出速看.md`，用于快速复习关键输出物和常见文档。

#### 场景：复习项目管理过程输出物

### 要求：每章必须预留案例归档位置
每个章节目录都必须包含 `案例/` 目录，用于放置真题、案例索引和相关材料。

#### 场景：整理章节案例
- **前提** 读者想把某章相关真题或案例材料归入对应章节

### 要求：核心项目管理章节必须提供输出物文件夹
第08章至第17章以及第19章的章节目录下必须包含 `输出物/` 目录，用于存放关键输出物样例文件。

#### 场景：查看核心章节的输出物样例
- **前提** 读者正在学习核心项目管理章节
- **当** 他进入第08章至第17章或第19章目录

- **并且** 问题既覆盖记忆性提问，也覆盖理解性和案例性追问
