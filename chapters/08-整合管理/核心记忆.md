<a id="core-mainline"></a>
## 一、整章主线

- 整合管理不是单独背一个知识点，而是把全项目拉到同一套节奏里。
- 你要把范围、进度、成本、变更、收尾放在一个闭环里看。
- 最核心主线：`授权 -> 计划 -> 执行 -> 监控 -> 变更 -> 收尾`
- 七过程口诀：`章 -> 计 -> 指 -> 知 -> 监 -> 变 -> 收`

<a id="core-basics"></a>
## 二、管理基础

### 1. 最该背

- 整合管理的责任主体是项目经理，整合这章本质上考的就是项目经理如何统筹全局。
- 整合管理目标：资源分配、平衡竞争需求、研究备选方法、裁剪过程、管理知识领域依赖。
- 执行整合的三个层面：过程层面、认知层面、背景层面。
- 项目复杂性：多个部分、相互关联、动态交互、整体大于部分相加。

### 2. 高频区分

- 项目管理计划偏“怎么管”。
- 项目文件偏“过程中形成了什么记录”。

<a id="core-charter"></a>
## 三、项目章程该记什么

- 先记住：项目章程不是模板题，而是考试里很爱追问的“授权文件”。
- 最少先背这 8 类内容：
  - 项目目的或立项依据
  - 可测量目标和成功标准
  - 高层级需求
  - 项目边界
  - 主要可交付成果
  - 总体风险
  - 里程碑进度计划
  - 预批准资源、项目经理权限、审批要求
- 你可以压缩成一句：
  `为什么做、做到什么、做哪些、不做哪些、交什么、有什么风险、何时完成、谁有权拍板`
- 高频提醒：
  - 章程是高层级，不写得像详细执行方案。
  - 章程解决“授权启动”，不是替代项目管理计划。

<a id="core-processes"></a>
## 四、七过程

### 1. 制定项目章程

- 关键词：正式批准项目，授权项目经理使用组织资源。
- 先记住：章程解决“能不能干、为什么干、谁牵头干”。
- 高频输出别漏：`项目章程 + 假设日志`
- 高频提醒：项目经理应尽量早任命，最好在制定章程时就明确。

展开阅读：
- [完整整理.md](./完整整理.md)
- [项目章程.md](./输出物/01-制定项目章程/项目章程.md)

### 2. 制订项目管理计划

- 关键词：整合所有规划成果。
- 最少先抓三大基准：范围基准、进度基准、成本基准。
- 计划组成别背漏：除了基准，还常包括`变更管理计划`、`配置管理计划`、`绩效测量基准`。
- 高频规则：基准确定前可反复细化；基准一旦确定，后续更新就要受控。

展开阅读：
- [完整整理.md](./完整整理.md)
- [项目管理计划.md](./输出物/02-制定项目管理计划/项目管理计划.md)

### 3. 指导与管理项目工作

- 关键词：按计划执行，并落实已批准变更。
- 高频输出：可交付成果、工作绩效数据、变更请求。
- 记流转：执行现场先产出`工作绩效数据`，这还不是正式报告。

展开阅读：
- [ITTO输出速看.md](./ITTO输出速看.md)
- [完整整理.md](./完整整理.md)

### 4. 管理项目知识

- 关键词：边做边沉淀。
- 高频文件：经验教训登记册，尽早建立、持续更新。

展开阅读：
- [完整整理.md](./完整整理.md)
- [经验教训登记册.md](./输出物/04-管理项目知识/经验教训登记册.md)

### 5. 监控项目工作

- 关键词：发现偏差、分析趋势、预测结果。
- 注意：监控不是直接改计划，先看状态、再判断是否需要走变更。
- 高频输出：`工作绩效报告 + 变更请求`
- 记流转：`工作绩效数据 -> 监控分析 -> 工作绩效报告 / 变更请求`

展开阅读：
- [完整整理.md](./完整整理.md)
- [工作绩效报告.md](./输出物/05-监控项目工作/工作绩效报告.md)

### 6. 实施整体变更控制

- 关键词：所有变更统一受控。
- 题干只要出现新需求、压工期、调预算、改范围，先想到整体变更控制。
- 高频输出：`批准的变更请求 + 变更日志`
- 记流转：`变更请求 -> 评审审批 -> 批准的变更请求 -> 执行落地`

展开阅读：
- [完整整理.md](./完整整理.md)
- [变更日志.md](./输出物/06-实施整体变更控制/变更日志.md)

### 7. 结束项目或阶段

- 关键词：验收、归档、总结、移交、释放资源。
- 收尾不是“做完就结束”，而是正式闭环。

展开阅读：

<a id="core-itto"></a>
## 五、七过程最小 ITTO 抓手

- 这部分不追求把全部 ITTO 一字不漏背下来，而是先抓“最常考输入 + 常用工具思路 + 高频输出”。

### 1. 制定项目章程

- 输入抓手：`商业文件 / 协议 / 事业环境因素 / 组织过程资产`
- 工具抓手：`专家判断 + 数据收集/人际沟通 + 会议`
- 输出抓手：`项目章程 + 假设日志`

### 2. 制订项目管理计划

- 输入抓手：`项目章程 + 其他过程输出 + 事业环境因素 + 组织过程资产`
- 工具抓手：`专家判断 + 数据收集 + 人际与团队技能 + 会议`
- 输出抓手：`项目管理计划`

### 3. 指导与管理项目工作

- 输入抓手：`项目管理计划 + 项目文件 + 批准的变更请求 + 事业环境因素 + 组织过程资产`
- 工具抓手：`专家判断 + 项目管理信息系统 + 会议`
- 输出抓手：`可交付成果 + 工作绩效数据 + 问题日志 + 变更请求`

### 4. 管理项目知识

- 输入抓手：`项目管理计划 + 项目文件 + 可交付成果 + 事业环境因素 + 组织过程资产`
- 工具抓手：`专家判断 + 知识管理 + 信息管理 + 人际与团队技能`
- 输出抓手：`经验教训登记册 + 项目管理计划更新 + 组织过程资产更新`

### 5. 监控项目工作

- 输入抓手：`项目管理计划 + 项目文件 + 工作绩效信息 + 协议 + 事业环境因素 + 组织过程资产`
- 工具抓手：`专家判断 + 数据分析 + 决策 + 会议`
- 输出抓手：`工作绩效报告 + 变更请求 + 项目管理计划更新 + 项目文件更新`

### 6. 实施整体变更控制

- 输入抓手：`项目管理计划 + 项目文件 + 工作绩效报告 + 变更请求 + 事业环境因素 + 组织过程资产`
- 工具抓手：`专家判断 + 变更控制工具 + 数据分析 + 决策 + 会议`
- 输出抓手：`批准的变更请求 + 变更日志 + 项目管理计划更新 + 项目文件更新`

### 7. 结束项目或阶段

- 输入抓手：`项目章程 + 项目管理计划 + 项目文件 + 验收的可交付成果 + 商业文件 / 协议 + 组织过程资产`
- 工具抓手：`专家判断 + 数据分析 + 会议`
- 输出抓手：`最终产品/服务/成果移交 + 项目最终报告 + 组织过程资产更新`

- 最小总记忆：
  - `章程` 吃业务理由，产授权文件
  - `计划` 吃各类规划成果，产总计划
  - `执行` 产成果、数据、问题、变更
  - `知识` 产经验教训
  - `监控` 产报告和调整信号
  - `变更` 产批准结果和变更台账
  - `收尾` 产移交、总结和归档

<a id="core-confusions"></a>
## 六、最易混点

- 项目章程 vs 项目管理计划
  章程解决“能不能干、谁牵头干”，计划解决“具体怎么干”。
- 项目管理计划 vs 项目文件
  计划是规则和依据，文件是记录和产物。
- 监控 vs 变更
  监控先发现偏差和问题，变更再决定怎么改。
- 批准的变更请求 vs 变更日志
  批准的变更请求是“批了可以改”，变更日志是“所有变更全过程留痕”。
- 配置控制 vs 变更控制
  配置控制更偏配置项状态受控，变更控制更偏对基准和可交付成果变化的统一评审。

<a id="core-review"></a>
## 七、案例题抓手

- 先判断项目有没有章程和计划，别一上来只写执行动作。
- 再顺着这条线答：`工作绩效数据 -> 工作绩效报告 -> 变更请求 -> 批准的变更请求 -> 执行`
- 如果题目给的是状态、数据、偏差、趋势，优先想监控项目工作。
- 如果题目给的是调整请求、基准变化、范围变化、工期变化，优先想整体变更控制。
- 最后别忘收尾闭环，案例题经常在“验收、归档、总结、移交”上埋坑。
