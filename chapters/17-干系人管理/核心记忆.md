<a id="core-mainline"></a>
## 一、整章主线

- 干系人管理不是一次性登记名单，而是持续识别、分类、拉动参与、监测变化。
- 最核心链条：`识别 -> 分类 -> 规划参与 -> 管理参与 -> 监督参与`
- 四过程口诀：`识 -> 规 -> 管 -> 监`
- 本章最常和沟通管理一起考，但两章关注点不一样。

<a id="core-basics"></a>
## 二、管理基础

### 1. 最该背

- 干系人管理的核心不是“知道他是谁”，而是“知道他想什么、能影响什么、怎么让他参与到位”。
- 干系人管理不是开头做一次，项目全过程都要动态更新。
- `权力利益方格` 是最常考的分类工具。
- `干系人参与度评估矩阵` 是最常考的参与管理工具。

### 2. 高频理解

- `干系人登记册` 偏“是谁、诉求是什么、影响力如何”。
- `干系人参与计划` 偏“怎么让他参与到位”。
- 当前参与度和期望参与度之间的差距，就是管理动作的抓手。

### 1. 识别干系人

- 先把关键干系人找出来，并初步分类排序。

### 2. 规划干系人参与

- 这一步是在定“希望谁以什么方式、在什么阶段参与到什么程度”。
- 高频输出：`干系人参与计划`
- 高频工具：`干系人参与度评估矩阵`

展开阅读：

### 3. 管理干系人参与

- 重点是把关系稳住，把支持拉起来，把阻力压下去。
- 高频提醒：这一步很看政治意识、沟通能力和关系管理。

展开阅读：

- [ITTO输出速看.md](./ITTO输出速看.md)

### 4. 监督干系人参与

- 持续看参与度有没有变化，策略还要不要调整。
- 高频输出：`工作绩效信息 + 变更请求 + 计划更新`

<a id="core-confusions"></a>
## 四、最易混点

- 沟通管理 vs 干系人管理
  一个更偏信息传递，一个更偏关系和参与度。
- 识别干系人 vs 规划干系人参与
  一个先找人并分类，一个再定参与策略。
- 管理干系人参与 vs 监督干系人参与
  一个偏推动参与，一个偏检查效果和调整。
- 当前参与度 vs 期望参与度
  一个是现状，一个是目标状态。

<a id="core-review"></a>
## 五、案例题抓手

- 如果题目说“关键人不支持、抵触、反复阻挠”，先看识别是否完整、分类是否准确、参与策略是否到位。
- 如果题目说“已经做了沟通但效果差”，别只写沟通管理，还要补干系人参与策略。
- 如果题目里某个干系人立场变化，记得写更新登记册、更新参与计划、重新管理参与。
- 本章很适合按“识别 -> 规划 -> 管理 -> 监督”顺序答。
