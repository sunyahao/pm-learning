# 工具与技术：挣值分析与常考计算白话说明

## 这份文档解决什么问题

很多人学成本管理，最大的问题不是不会按公式按计算器，而是：

`算出来了，但不知道结果到底说明项目现在是省钱了、超支了、提前了、滞后了，还是后面根本守不住预算。`

这份文档就是专门把第11章最常考的计算讲成“会算 + 会判 + 会说”。

## 为什么这些缩写是这么来的

如果你只背中文，很容易越背越乱。  
其实这些缩写大多就是把英文关键词首字母拿出来，所以知道原词后会更好记。

### 1. PV

- 全称：`Planned Value`
- 拆开记：
  - `P = Planned`，计划的
  - `V = Value`，价值
- 所以 `PV` 就是：`计划值`

### 2. EV

- 全称：`Earned Value`
- 拆开记：
  - `E = Earned`，挣到的、赚到的
  - `V = Value`，价值
- 所以 `EV` 就是：`挣值`

这里最容易卡的是：为什么不是 `AV`？

因为这个词强调的不是“实际发生了什么”，而是“真正挣到了多少价值”。  
它关注的是你到底完成了多少有价值的工作。

### 3. AC

- 全称：`Actual Cost`
- 拆开记：
  - `A = Actual`，实际的
  - `C = Cost`，成本
- 所以 `AC` 就是：`实际成本`

### 4. CV 和 SV

- `CV = Cost Variance`
- `SV = Schedule Variance`

这里的关键是：

- `C = Cost`，成本
- `S = Schedule`，进度
- `V = Variance`，偏差

所以：

- `CV`：成本偏差
- `SV`：进度偏差

### 5. CPI 和 SPI

- `CPI = Cost Performance Index`
- `SPI = Schedule Performance Index`

这里的关键是：

- `C = Cost`
- `S = Schedule`
- `P = Performance`
- `I = Index`

所以：

- `CPI`：成本绩效指数
- `SPI`：进度绩效指数

### 6. EAC、ETC、VAC

- `EAC = Estimate at Completion`
- `ETC = Estimate to Complete`
- `VAC = Variance at Completion`

拆开记：

- `E = Estimate`，估算
- `V = Variance`，偏差
- `Completion / Complete` 都和“完工、完成”有关

所以：

- `EAC`：完工时的总估算
- `ETC`：从现在到完成还要多少
- `VAC`：到完工时相对预算的偏差

### 7. TCPI

- 全称：`To-Complete Performance Index`

拆开记：

- `To Complete`：为了完成剩余工作
- `Performance Index`：需要达到的绩效指数

所以 `TCPI` 的意思就是：

`后半程必须跑出多高的效率，才能守住目标`

## 记缩写时最好用的总规律

### 先记几个高频词根

- `V` 常见有两种：
  - `Value`：价值
  - `Variance`：偏差
- `C` 常见有两种：
  - `Cost`：成本
  - `Completion / Complete`：完工、完成
- `S` 常见是：
  - `Schedule`：进度
- `P` 常见有两种：
  - `Planned`：计划的
  - `Performance`：绩效
- `I` 常见是：
  - `Index`：指数
- `E` 常见有两种：
  - `Earned`：挣到的
  - `Estimate`：估算
- `A` 常见有两种：
  - `Actual`：实际的
  - `At`：在……时

### 再按“同类成组”去记

- 值：`PV / EV / AC`
- 偏差：`CV / SV / VAC`
- 指数：`CPI / SPI / TCPI`
- 预测：`EAC / ETC`

这样会比一条条硬背轻松很多。

## 先把三个基础量吃透

### 1. PV

- 全称：`Planned Value`
- 中文：`计划值`
- 大白话：按原计划，到现在本来应该完成多少价值
- 你可以理解成：`计划里今天本该挣到的钱`

### 2. EV

- 全称：`Earned Value`
- 中文：`挣值`
- 大白话：到现在实际上真正完成了多少价值
- 你可以理解成：`项目真实挣到的钱`

### 3. AC

- 全称：`Actual Cost`
- 中文：`实际成本`
- 大白话：为了做到现在这个进度，实际已经花了多少钱
- 你可以理解成：`已经真花出去的钱`

## 第一层判断：先别急着算指数，先看三个基础量

最常见的比较方式：

- `EV < PV`：说明实际完成的价值比计划少，通常表示进度落后
- `EV > PV`：说明实际完成的价值比计划多，通常表示进度提前
- `EV < AC`：说明花出去的钱比挣到的价值多，通常表示成本超支
- `EV > AC`：说明挣到的价值比花的钱多，通常表示成本节约

你可以先记一句：

- `EV 跟 PV 比，看进度`
- `EV 跟 AC 比，看成本`

## 第二层判断：最常考的四个指标

### 1. CV：成本偏差

- 公式：`CV = EV - AC`
- 大白话：现在是省钱了还是超支了
- 判断：
  - `CV > 0`：成本节约
  - `CV < 0`：成本超支
- 一句话记忆：`挣到的比花的多，就是省；挣到的比花的少，就是超`

### 2. SV：进度偏差

- 公式：`SV = EV - PV`
- 大白话：现在是比原计划快还是慢
- 判断：
  - `SV > 0`：进度提前
  - `SV < 0`：进度落后
- 一句话记忆：`实际挣到的比原计划多，就是快；比原计划少，就是慢`

### 3. CPI：成本绩效指数

- 公式：`CPI = EV / AC`
- 大白话：每花 1 块钱，换回了多少价值
- 判断：
  - `CPI > 1`：成本效率好
  - `CPI < 1`：成本效率差
- 一句话记忆：`花 1 块能挣回超过 1 块价值，就是划算`

### 4. SPI：进度绩效指数

- 公式：`SPI = EV / PV`
- 大白话：实际推进速度和计划速度比怎么样
- 判断：
  - `SPI > 1`：进度效率好
  - `SPI < 1`：进度效率差
- 一句话记忆：`实际跑得比计划快，SPI 就大于 1`

## 第三层判断：后面会怎么样

### 1. EAC：完工估算

- 全称：`Estimate at Completion`
- 大白话：照现在这个势头，项目做完一共要花多少钱

常考写法有几种，你先抓最常见的：

- `EAC = BAC / CPI`
  - 适合：默认当前成本绩效会持续下去
- `EAC = AC + ETC`
  - 适合：后面的工作重新估，不完全按当前效率延续

一句话记忆：

- `EAC` 看结局总额

### 2. ETC：完工尚需估算

- 全称：`Estimate to Complete`
- 大白话：从现在做到项目结束，还得再花多少钱
- 常见公式：`ETC = EAC - AC`

一句话记忆：

- `ETC` 看后面还要花多少

### 3. VAC：完工偏差

- 全称：`Variance at Completion`
- 大白话：项目最后相对原预算会差多少
- 公式：`VAC = BAC - EAC`
- 判断：
  - `VAC > 0`：预计还能省
  - `VAC < 0`：预计最后会超

一句话记忆：

- `VAC` 看最后是剩钱还是差钱

### 4. TCPI：完工尚需绩效指数

- 全称：`To-Complete Performance Index`
- 大白话：从现在开始，后面得多高效，才能守住原预算或新预算

常见公式：

- 基于原预算：`TCPI = (BAC - EV) / (BAC - AC)`
- 基于新完工估算：`TCPI = (BAC - EV) / (EAC - AC)` 或教材变体按目标值理解

你先抓意思：

- `TCPI > 1`：后面必须比现在更高效，压力大
- `TCPI < 1`：后面要求相对没那么紧

一句话记忆：

- `TCPI` 看后半场要不要拼命追

## 一道完整例子，串着看

假设题目给你：

- `BAC = 200 万`
- `PV = 100 万`
- `EV = 80 万`
- `AC = 120 万`

### 第一步：先看项目状态

- `EV < PV`
  - 80 小于 100
  - 说明进度落后
- `EV < AC`
  - 80 小于 120
  - 说明成本超支

这时候就已经能先写一句判断：

`项目当前进度落后，且成本超支。`

### 第二步：再算偏差和指数

- `CV = EV - AC = 80 - 120 = -40`
  - 成本超支 40 万
- `SV = EV - PV = 80 - 100 = -20`
  - 进度落后 20 万价值量
- `CPI = EV / AC = 80 / 120 = 0.67`
  - 成本绩效偏低
- `SPI = EV / PV = 80 / 100 = 0.8`
  - 进度绩效偏低

### 第三步：再看后面大概率会怎样

如果默认当前成本效率会持续：

- `EAC = BAC / CPI = 200 / 0.67 ≈ 298.51 万`

那就说明：

`如果不纠偏，项目完工总成本可能接近 299 万，明显高于原预算 200 万。`

再算：

- `ETC = EAC - AC ≈ 298.51 - 120 = 178.51 万`
- `VAC = BAC - EAC ≈ 200 - 298.51 = -98.51 万`

这就能继续写：

`预计项目最终将超预算约 98.51 万。`

## 做题时最稳的答题顺序

1. 先抄清题目给的已知量
2. 先比较 `EV`、`PV`、`AC`
3. 再算 `CV`、`SV`、`CPI`、`SPI`
4. 最后再算 `EAC`、`ETC`、`VAC`、`TCPI`
5. 每算完一步，都用一句人话解释结果

最容易丢分的地方不是计算错，而是：

- 只写公式结果，不写项目状态
- 把 `CV` 和 `SV` 搞反
- 把 `CPI` 和 `SPI` 搞反
- `> 1`、`< 1` 的意义背反

## 你一定要会的口头翻译

如果题里让你“分析项目当前状态”，你不要只写：

- `CPI = 0.82`
- `SPI = 0.91`

你要继续翻译成人话：

- `CPI = 0.82，说明项目成本绩效较差，存在超支`
- `SPI = 0.91，说明项目进度绩效较差，存在落后`
- `如果不采取纠偏措施，项目最终完工成本可能高于原预算`

## 一页速记

- `PV`：本来该完成多少
- `EV`：实际上完成多少
- `AC`：实际上花了多少
- `CV = EV - AC`：看超支还是节约
- `SV = EV - PV`：看提前还是落后
- `CPI = EV / AC`：看花钱效率
- `SPI = EV / PV`：看进度效率
- `EAC`：最后总共要花多少
- `ETC`：后面还得花多少
- `VAC = BAC - EAC`：最后差预算多少
- `TCPI`：后面得多高效才守得住目标
