# 精讲：第11章 成本管理

## 来源定位

- 对应 PDF 章节：第11章-成本管理
- 对应页码范围：43-45
- 这份精讲在本章来源基础上补充了教学式串联，重点帮你把“估算、预算、控制、挣值分析”真正连成一套管理语言。

## 先把这章想明白

成本管理不是单纯算公式，而是在回答三个问题：

1. 这个项目大概要花多少钱。
2. 这些钱该怎样形成正式预算基准。
3. 项目进行过程中，当前花钱状态到底健康不健康。

很多人学成本管理时，只会背 EV、PV、AC 和一堆公式，却不会解释项目现在是超支还是节约，是提前还是滞后，后续还要不要纠偏。这种学法很容易丢分。

真正学透成本管理，要做到三步：

- 会算
- 会判
- 会解释

## 四过程是一条从估算走向控制的链

### 1. 规划成本管理

先确定成本以后怎么管。

- 关键输出是成本管理计划。
- 这一层主要解决方法、口径、精度、控制方式，而不是直接给出具体预算数。

### 2. 估算成本

这一过程解决“各项活动或工作包大概要花多少钱”。

常见输出有：

- 成本估算
- 估算依据

这里你要特别注意一个认知：

- 成本估算不是成本基准。
- 估算更像分项金额和测算结果。
- 基准则是后续拿来正式比较偏差的控制基线。

### 3. 制定预算

这一过程是把各项成本估算汇总、平衡、上升为项目的正式资金安排。

关键输出包括：

- 成本基准
- 项目资金需求

成本基准是考试高频词。你可以把它理解成：

- 这是项目成本控制的正式参照线。
- 以后算偏差、做预测、提变更，都会不断回到基准。

同时，你要把两类储备分清：

- 应急储备：针对已识别风险准备的储备。
- 管理储备：针对未知风险或管理层预留的储备。

这组区分在选择题里很爱出。

### 4. 控制成本

这一过程是成本管理最核心、最常出计算题的地方。

关键输出包括：

- 工作绩效信息
- 成本预测
- 变更请求
- 相关计划和文件更新

这里最核心的工具就是挣值分析。

## 挣值分析要学成“管理判断”，不是公式背诵

### 三个基础量

- PV：计划值，原来计划到现在应该完成多少价值。
- EV：挣值，实际已经完成了多少价值。
- AC：实际成本，已经实际花了多少钱。

这三个量是所有后续判断的基础。

### 四个高频判断指标

- CV = EV - AC  
  看成本偏差。
- SV = EV - PV  
  看进度偏差。
- CPI = EV / AC  
  看成本绩效效率。
- SPI = EV / PV  
  看进度绩效效率。

最常见判断口诀是：

- CV > 0：节约
- CV < 0：超支
- SV > 0：提前
- SV < 0：滞后
- CPI > 1：成本效率好
- CPI < 1：成本效率差
- SPI > 1：进度效率好
- SPI < 1：进度效率差

但考试真正拉分的地方，不在于你会不会背这些符号，而在于你能不能把结果翻译成项目状态描述。

### 预测类指标

本章还会碰到 EAC、ETC、TCPI 等预测指标。

最重要的不是死背所有变体，而是明白它们在回答什么：

- EAC：项目完工时预计总成本会是多少。
- ETC：从现在开始到做完还要花多少。
- TCPI：后续必须达到怎样的成本绩效水平，才能守住原目标或新目标。

所以当题目给出一串数字时，你先不要慌着套公式，先判断它到底是在问“现在状态”，还是在问“后面预测”。

## 本章最重要的三组区分

### 成本估算 vs 成本基准

- 成本估算偏测算结果。
- 成本基准偏正式控制依据。

### 应急储备 vs 管理储备

- 应急储备针对已识别风险。
- 管理储备针对未知事项或管理层预留。

### 成本偏差 vs 进度偏差

- 成本偏差看花钱状态。
- 进度偏差看时间推进状态。

很多题故意把 CPI、SPI、CV、SV 混着给，就是为了看你会不会在高压下把概念分清。

## 为什么很多人“会算不会答”

因为他只停留在数字层面，没有转成管理语言。

比如你算出 CPI = 0.8，真正好的答案不该只写“CPI 小于 1”，而要继续写：

- 当前成本绩效不佳；
- 项目已经超支；
- 若不采取纠偏措施，完工成本可能高于原预算；
- 应结合原因分析、预测和变更控制采取调整措施。

这才是考试想要的项目管理表达。

## 案例题怎么组织语言

常见答题抓手有四个：

- 是否先完成成本估算并形成成本基准
- 是否按计划监控成本执行情况
- 是否结合挣值数据进行偏差和趋势分析
- 成本失控时是否进行了预测、报告和变更处理

如果题里给了数字，你就按“先算 -> 再判 -> 最后解释影响和措施”的顺序答。

## 最后的学习提醒

成本管理真正要学会的，不是把公式抄对，而是把数字背后的项目状态讲明白。

你最后一定要能顺着这条链讲下来：

估算多少钱 -> 汇总成什么预算基准 -> 当前花费和产出比怎么样 -> 后续会不会超 -> 要不要纠偏和变更

这条链顺了，成本管理就不再只是“算题章”。
