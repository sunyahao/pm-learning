### 3. 定义范围

主要输出物：

- [项目范围说明书.md](/D:/codex-project/chapters/09-范围管理/输出物/03-定义范围/项目范围说明书.md)

这一过程解决的是：

- 项目具体做什么
- 不做什么
- 验收时按什么标准说“做完了”

### 4. 创建 WBS

主要输出物：

- [WBS.md](/D:/codex-project/chapters/09-范围管理/输出物/04-创建WBS/WBS.md)
- [WBS词典.md](/D:/codex-project/chapters/09-范围管理/输出物/04-创建WBS/WBS词典.md)

补充说明：

- 这一过程在教材里正式输出是 `范围基准`
- 范围基准通常可以理解成三件套：`项目范围说明书 + WBS + WBS词典`

这一过程解决的是：

- 把已经定义好的范围继续拆成结构化工作包
- 让后续进度、成本、资源管理有共同基础

### 5. 确认范围

主要输出物样例：

- [验收确认单.md](/D:/codex-project/chapters/09-范围管理/输出物/05-确认范围/验收确认单.md)

补充说明：

- 教材里这一过程还会产生 `验收的可交付成果`、`工作绩效信息`、`变更请求`
- 这里用 `验收确认单` 来帮助理解“正式确认范围”在项目里长什么样

这一过程解决的是：

- 不是项目组自己觉得做完，而是干系人正式确认“这就是我要的”

### 6. 控制范围

主要输出物样例：

- [变更请求单.md](/D:/codex-project/chapters/09-范围管理/输出物/06-控制范围/变更请求单.md)

补充说明：

- 教材里这一过程还会产生 `工作绩效信息`、`项目管理计划更新`、`项目文件更新`
- 这里用 `变更请求单` 来帮助理解范围变动时，项目里是怎么正式提出和处理变化的

这一过程解决的是：

- 防止范围蔓延
- 让新增、删除和调整内容进入正式控制
