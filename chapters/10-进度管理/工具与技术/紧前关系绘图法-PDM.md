# 紧前关系绘图法 PDM（样例）

## 这是什么

`PDM` 是 `Precedence Diagramming Method`，常译为 `紧前关系绘图法`。

它的特点是：

- `活动写在节点里`
- 节点之间用箭线表示逻辑关系
- 最常见的是 `FS`，但也能直接表示 `SS`、`FF`、`SF`

所以它比 `ADM` 更灵活，也是高项里更常见的网络图表达方式。

## 最小可读样例

假设我们有下面几个活动：

- A：完成需求评审
- B：接口开发
- C：前端开发
- D：联调测试
- E：上线准备

其中关系如下：

- A 完成后，B 才能开始
- A 完成后，C 才能开始
- B 和 C 都完成后，D 才能开始
- D 完成后，E 才能开始

可以画成：

```text
+------------------+      FS      +------------------+
| A 需求评审       | -----------> | B 接口开发       |
+------------------+              +------------------+
         |                                  |
         | FS                               | FS
         v                                  v
+------------------+              +------------------+
| C 前端开发       | -----------> | D 联调测试       |
+------------------+      FS      +------------------+
                                            |
                                            | FS
                                            v
                                   +------------------+
                                   | E 上线准备       |
                                   +------------------+
```

更准确地表示 “B 和 C 都完成后，D 才能开始”，可以理解为：

- `B -> D` 是一条 `FS`
- `C -> D` 也是一条 `FS`

## 四种关系在 PDM 里怎么长

### 1. FS：完成到开始

最常见。

```text
[A 完成需求评审] ----FS----> [B 接口开发]
```

意思是：A 不完成，B 不能开始。

### 2. SS：开始到开始

```text
[A 测试准备] ----SS----> [B 培训材料编写]
```

意思是：A 一开始，B 就可以开始，不用等 A 全部完成。

### 3. FF：完成到完成

```text
[A 联调执行] ----FF----> [B 联调记录整理]
```

意思是：A 不完成，B 也不能完成。

### 4. SF：开始到完成

比较少见。

```text
[A 新系统切换开始] ----SF----> [B 旧系统值守结束]
```

意思是：A 一开始，B 才能结束。

## 放到日常项目里怎么理解

PDM 最适合表达：

- 哪些事必须等前面做完
- 哪些事可以并行启动
- 哪些事结束要受别的活动约束

比如政务平台项目里：

- 需求评审后才能开发
- 测试一开始，培训材料也可以同步准备
- 新系统切换开始后，旧系统值守才能退出

这些关系用 PDM 一眼就能看出来。

## 为什么考试爱考它

- 因为它能直接结合 `FS/SS/FF/SF`
- 容易和 `提前量`、`滞后量` 联动考
- 后续还能继续推到 `关键路径`

## 最容易混淆的点

- `PDM` 是活动在节点里，不是活动写在箭线上。
- `PDM` 不等于甘特图。
- `PDM` 重点是逻辑关系，不是开始结束日期本身。
