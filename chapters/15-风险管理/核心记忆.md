<a id="core-mainline"></a>
## 一、整章主线

- 风险管理不是出问题后补救，而是在问题发生前先识别、先评估、先准备。
- 最核心链条：`识别风险 -> 定性/定量分析 -> 规划应对 -> 落实应对 -> 持续监督`
- 七过程口诀：`规 -> 识 -> 定性 -> 定量 -> 应对 -> 实施 -> 监`
- 本章很爱考“风险和问题”“登记册和报告”“储备怎么用”。

<a id="core-basics"></a>
## 二、管理基础

### 1. 最该背

- 风险是`不确定事件或条件`，问题是`已经发生的事实`。
- 已知风险更适合配 `应急储备`。
- 未知风险通常通过 `管理储备` 覆盖。
- 风险既可能带来威胁，也可能带来机会。

### 2. 高频文件和工具

- `RBS`：风险分解结构
- `SWOT`：优势、劣势、机会、威胁
- `风险登记册` 偏单个风险明细台账。
- `风险报告` 偏整体风险概况和趋势。

## 三、七过程

### 1. 规划风险管理

- 先把风险怎么识别、怎么评估、怎么应对、怎么汇报定下来。
- 高频输出：`风险管理计划`

### 2. 识别风险

- 这一步是把项目里可能出事或带来机会的点尽量找出来。
- 高频输出：`风险登记册 + 风险报告`

### 3. 实施定性风险分析

- 先按概率、影响、紧迫性做快速筛选和排序。
- 不是所有风险都值得进入定量分析。

- 这一步更偏数字化分析，不是所有项目都必须做。
- 常见工具：模拟、敏感性分析、决策树分析。

展开阅读：

- 先根据威胁、机会、整体项目风险分别定策略。
- 高频输出：`变更请求 + 计划更新 + 文件更新`

### 6. 实施风险应对

- 重点不是“有方案”，而是“方案真的落地了”。
- 这一步很容易在案例题里被漏写。

展开阅读：

- 持续看风险状态有没有变化、储备够不够、应对策略还适不适合。
- 高频工具：储备分析、风险审计、技术绩效分析。

<a id="core-confusions"></a>
## 四、最易混点

- 风险 vs 问题
  一个是未发生的不确定性，一个是已经发生的事实。
- 风险登记册 vs 风险报告
  一个偏单个风险明细，一个偏整体风险总览。
- 定性分析 vs 定量分析
  一个偏排序筛选，一个偏数字测算。
- 应急储备 vs 管理储备
  一个应对已知风险，一个应对未知风险。
- 威胁应对 vs 机会应对
  一个是防坏事，一个是抓好事。

<a id="core-review"></a>
## 五、案例题抓手

- 案例题一旦写到风险，先按“识别 -> 分析 -> 应对 -> 监督”排动作。
- 如果题目里风险已经发生了，就别再写“识别风险”，要转成问题处理和后续风险控制。
- 如果题目说储备不够或预算被动用，先区分是应急储备还是管理储备。
- 这章经常和成本、进度、采购联动考。
