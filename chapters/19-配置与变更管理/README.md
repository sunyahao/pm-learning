## 本章定位

- 本章是“基线受控”和“变化受控”的关键章节，常和整合管理、采购管理、成本控制、收尾与文档管理一起考。

## 核心考点

- 配置项、状态、版本号、基线、配置库三者的关系。
- 配置管理活动：计划、识别、控制、状态报告、审计、回顾改进。
- 变更原因、分类、角色与八步闭环：申请、初审、论证、审查、实施、监控、评估、收尾。
- 版本发布、回退机制、配置状态留痕与文档规范化管理。

## 高频关键词

- 配置项、基线、配置库、配置审计、CCB、变更初审、变更论证、版本发布、回退、文档管理。

## 易错点

- 容易把配置管理活动和变更管理活动混成一套动作。
- 容易只写审批，不写初审、论证、监控、评估和收尾。
- 容易忽略配置库更新、版本保留、回退方案和状态报告。

## 复盘提示

- 这章要按“配置受控 -> 变更闭环 -> 发布回退 -> 归档留痕”顺序理解。
