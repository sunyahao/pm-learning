# 项目管理统合理解

这个目录给“已经把十大知识领域都看过一遍，但开始记忆和刷题时还是觉得散”的学习者使用。

它不替代各章的 `README.md`、`主脉络.md`、`核心记忆.md`，而是补一层跨章节、跨过程组、跨 ITTO 的总览视角。

## 这个目录解决什么问题

- 十大知识领域和五大过程组怎么整体串起来。
- 为什么很多过程都反复出现 `事业环境因素`、`组织过程资产`、`专家判断`、`数据收集`、`会议`。
- 做题时怎么先从“过程组 + 共性 ITTO”建立判断，再回到具体知识领域。

## 阅读顺序

1. 先看 [十大知识领域与五大过程组交叉矩阵.md](./十大知识领域与五大过程组交叉矩阵.md)
2. 再看 [事业环境因素与组织过程资产及通用工具技术.md](./事业环境因素与组织过程资产及通用工具技术.md)
3. 再回到对应章节的 `主脉络.md`、`核心记忆.md`、`复盘题单.md`

## 适合什么时候看

- 你已经不再是“完全没看过”，但脑子里还是串不起来。
- 你发现自己会背过程名，却不知道为什么很多 ITTO 长得像。
- 你刷案例题时，总分不清这是在考范围、进度、成本，还是在考共性的管理动作。

## 和其他 guides 怎么配合

- 想先把 24 章主线串起来，看 [24章串联学习地图](../24章串联学习地图.md)
- 想知道每天按什么顺序看章节资产，看 [章节记忆导航](../章节记忆导航.md)
- 想知道当前仓库的提分重点在哪，看 [资料盘点与提分优化建议](../资料盘点与提分优化建议.md)
