# 事业环境因素、组织过程资产与通用工具技术总览

这份文档解决两个常见困惑：

1. 为什么很多过程的输入里总会出现 `事业环境因素` 和 `组织过程资产`。
2. 为什么很多过程的工具与技术里总会反复出现 `专家判断`、`数据收集`、`人际关系与团队技能`、`会议`。

如果你把这些内容看成“重复项”，会越背越乱；  
如果你把它们看成“跨过程共用母题”，就会越来越好记。

## 一、先抓总逻辑

- `事业环境因素`：项目外部或组织当前环境里，客观存在、项目团队通常不能随意改的条件。
- `组织过程资产`：组织已经沉淀下来的模板、流程、经验、知识库和历史资料。
- `工具与技术`：项目经理和团队在执行过程时，用来判断、分析、沟通、组织、决策的办法。

你可以先这样压缩：

- `事业环境因素` 像环境和土壤
- `组织过程资产` 像组织给你的旧资料和旧工具箱
- `工具与技术` 像你这次做题、做项目时手里真的会用到的方法

## 二、什么是事业环境因素

### 1. 大白话理解

事业环境因素不是你这次项目自己发明出来的，而是项目启动前就已经在那里了。

它们会影响你怎么授权、怎么沟通、怎么审批、怎么采购、怎么汇报、怎么用系统，但项目团队通常不能说改就改。

### 2. 常见内容

- 组织文化和管理风格
- 治理框架、审批规则、权限边界
- 市场环境、行业竞争、供应商现状
- 法律法规、标准规范、合同环境
- 信息系统条件、项目管理信息系统
- 基础设施、地域分布、时区、办公条件
- 组织现有的人力资源状况

### 3. 为什么很多过程都把它当输入

因为项目管理不是在真空里发生的。

无论你在做章程、计划、采购、沟通、风险还是收尾，都会受到当前组织环境和外部环境影响。

### 4. 记忆抓手

- 只要它更像“环境约束、现实条件、制度背景”，优先往 `事业环境因素` 想。
- 只要它是“这次项目没法随意改，但又必须受它影响”，也优先往 `事业环境因素` 想。

一句话记忆：
`事业环境因素，就是项目所处的现实环境和客观条件。`

## 三、什么是组织过程资产

### 1. 大白话理解

组织过程资产是组织以前干过项目后留下来的“可复用家底”。

它们不是环境本身，而是组织已经积累好的模板、规范、经验和知识。

### 2. 常见内容

- 规章制度、标准流程、作业程序
- 模板、表单、检查单、审批单
- 历史项目档案、经验教训、知识库
- 风险分类、估算历史数据、基准数据
- 变更流程资料、质量标准、合同样板

### 3. 为什么很多过程都把它当输入

因为项目管理经常不是从零开始干，而是站在组织既有经验上继续做。

比如你做章程、计划、范围、风险、采购、收尾时，常常都会参考以前项目的模板、教训和记录。

### 4. 记忆抓手

- 只要它更像“组织已经留好的模板、制度、历史资料”，优先往 `组织过程资产` 想。
- 只要它带有“可复用、可继承、可参考”的味道，也优先往 `组织过程资产` 想。

一句话记忆：
`组织过程资产，就是组织以前攒下来的模板、制度和经验家底。`

## 四、怎么区分事业环境因素和组织过程资产

最实用的判断法：

- 更像“外部条件、现实环境、当前约束”，是 `事业环境因素`
- 更像“组织沉淀、历史资料、内部模板”，是 `组织过程资产`

对比例子：

- 组织文化、审批层级、法规要求：`事业环境因素`
- 章程模板、风险清单、经验教训库：`组织过程资产`
- 供应商市场现状：`事业环境因素`
- 采购评分模板、合同样板：`组织过程资产`
- 现有 PMIS 系统环境：通常更偏 `事业环境因素`
- 历史项目数据报表：更偏 `组织过程资产`

## 五、为什么这些工具与技术总是重复出现

很多过程虽然主题不同，但项目经理做事时底层动作其实就那几类：

- 需要借助经验判断
- 需要收集信息
- 需要拉人沟通
- 需要开会形成共识
- 需要分析与决策

所以你会反复看到相似的工具与技术。

这不是“教材偷懒”，而是项目管理底层方法本来就跨过程复用。

## 六、按相同维度梳理通用工具与技术

### 1. 判断类：专家判断

- 大白话：自己拿不准，或事情本来就需要经验和专业意见时，就要借专家。
- 为什么高频出现：
  - 立项要判断是否值得做
  - 计划要判断怎么编更合理
  - 风险要判断概率和影响
  - 变更要判断影响多大
  - 收尾要判断是否真正达到目标
- 一句话记忆：
  `凡是需要专业经验、跨领域判断、拍板取舍时，常会出现专家判断。`

### 2. 收集类：数据收集

这类工具本质上是在回答“信息从哪里来”。

#### 头脑风暴

- 适合：先把想法尽量放出来
- 常见用途：识别风险、收集需求、寻找方案
- 记忆抓手：
  `先求多，再整理`

#### 核对单

- 适合：防漏项
- 常见用途：风险识别、质量检查、收尾核查
- 记忆抓手：
  `先有清单，再逐项对`

#### 焦点小组

- 适合：围绕某一主题让一组典型代表集中讨论
- 常见用途：需求理解、产品方向、用户看法
- 记忆抓手：
  `小范围、集中聊、挖观点`

#### 访谈

- 适合：针对关键人深入了解信息
- 常见用途：章程、需求、干系人、经验收集
- 记忆抓手：
  `一对一或小范围，挖深度`

### 3. 人际与团队类：把事拉通

这类工具本质上是在回答“人怎么协同”。

#### 冲突管理

- 适合：资源争抢、目标冲突、意见对立
- 为什么高频出现：
  项目里多方目标常常不一致，不处理冲突，流程再对也推不动。
- 记忆抓手：
  `冲突不是异常，而是项目常态。`

#### 引导

- 适合：会议跑偏、观点发散、多人难达成一致
- 为什么高频出现：
  很多过程不是项目经理自己写完就行，而是要把大家拉到同一认识上。
- 记忆抓手：
  `引导就是让讨论从发散走向共识。`

### 4. 协同类：会议

- 大白话：很多管理动作最终都要通过会议来达成统一理解、统一判断、统一决定。
- 为什么高频出现：
  - 章程要对齐授权和目标
  - 计划要协调范围、进度、成本
  - 变更要评估影响并审批
  - 收尾要确认移交和总结
- 记忆抓手：
  `只要涉及多人对齐、协商、拍板，会议都很常见。`

### 5. 分析与决策类

虽然你这次先问的是 `专家判断、数据收集、人际关系与团队技能、会议`，但刷题时还经常会碰到分析与决策类工具，也建议一起记。

- 数据分析：
  用来找偏差、看趋势、评估影响、预测结果
- 决策：
  用来在多个备选方案之间做取舍

一句话记忆：
`收集是拿到材料，分析是看懂材料，决策是选出方案。`

## 七、怎么把这些共性 ITTO 套回十大知识领域

你可以用下面这套顺序去看任何过程：

1. 这个过程先受什么环境和制度影响
2. 组织以前有没有模板、经验、历史资料可参考
3. 需要找谁来判断或提供信息
4. 需要通过什么沟通方式把人拉齐
5. 最后形成什么文件、报告、决定或更新

这样看，你就不会只是在背 ITTO 名称，而是在理解一个过程为什么会“长成这样”。

## 八、最短总记忆

- `事业环境因素`：现实环境和客观条件
- `组织过程资产`：组织沉淀的模板与经验
- `专家判断`：经验型判断与拍板
- `数据收集`：把信息先拿全
- `人际与团队技能`：把人拉通
- `会议`：把共识和决定落地

最后压成一句：

`环境决定你受什么影响，资产决定你能借什么家底，工具决定你这次怎么把事做成。`
