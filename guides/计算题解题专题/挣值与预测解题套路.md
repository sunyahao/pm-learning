# 挣值与预测解题套路

## 这类题到底在考什么

这类题表面上在考公式，实际上在考三件事：

1. 项目现在是超前还是落后
2. 项目现在是节约还是超支
3. 按当前趋势，后面会怎样

所以它不是纯数学题，而是“数字版项目状态判断题”。

## 一、先认题型信号

题干如果出现这些词，基本就是挣值题：

- BAC
- EV、PV、AC
- CV、SV
- CPI、SPI
- EAC、ETC、VAC、TCPI

## 二、最稳的解题顺序

### 第一步：先认三大基础量

- `PV`：按计划现在应该完成多少价值
- `EV`：实际完成了多少价值
- `AC`：为了做到现在这个程度，实际花了多少钱

如果这三个量都认不稳，后面全乱。

### 第二步：先算偏差

- `SV = EV - PV`
- `CV = EV - AC`

判断最直观：

- `SV < 0`：进度落后
- `CV < 0`：成本超支

### 第三步：再算指数

- `SPI = EV / PV`
- `CPI = EV / AC`

判断最常用：

- `SPI < 1`：进度效率差
- `CPI < 1`：成本效率差

### 第四步：最后看预测

最常见的预测量：

- `EAC`：照当前趋势，完工时总共要花多少
- `ETC`：从现在到完工还要花多少
- `VAC`：最终会比原预算多还是少

## 三、最常见的坑

### 坑 1：把 EV 和 AC 混掉

- `EV` 是成果价值
- `AC` 是实际花费

### 坑 2：只会算，不会翻译

比如 `CPI = 0.73`，不能只写“0.73”，要写“成本效率偏低，项目当前超支”。

### 坑 3：预测题一上来乱套公式

先问自己：

- 题目是在问总共要花多少？`EAC`
- 还是问后面还要花多少？`ETC`
- 还是问最后会差多少？`VAC`

## 四、例题拆解

### 题目

某项目预算总额 `BAC = 100 万元`。

到第 5 个月末：

- 按计划应完成 50%
- 实际完成 40%
- 实际已花费 55 万元

求：

1. `PV`、`EV`、`AC`
2. `SV`、`CV`
3. `SPI`、`CPI`
4. 若按当前成本绩效持续下去，`EAC` 和 `ETC` 各是多少
5. 说明项目当前状态

### 解题

#### 1. 基础量

- `PV = 100 × 50% = 50`
- `EV = 100 × 40% = 40`
- `AC = 55`

#### 2. 偏差

- `SV = EV - PV = 40 - 50 = -10`
- `CV = EV - AC = 40 - 55 = -15`

判断：

- `SV < 0`，进度落后
- `CV < 0`，成本超支

#### 3. 指数

- `SPI = EV / PV = 40 / 50 = 0.80`
- `CPI = EV / AC = 40 / 55 ≈ 0.73`

判断：

- `SPI < 1`，进度效率偏低
- `CPI < 1`，成本效率偏低

#### 4. 预测

若题干假设“后续仍按当前成本效率推进”，常用：

- `EAC = BAC / CPI = 100 / 0.73 ≈ 137.5`
- `ETC = EAC - AC = 137.5 - 55 = 82.5`

#### 5. 结论

- 当前项目进度落后、成本超支
- 如果不纠偏，预计完工总成本约为 `137.5 万元`
- 从现在到结束还需约 `82.5 万元`

## 五、案例题里怎么写得更稳

不要只写计算结果，建议按这个顺序写：

1. 先写当前状态：进度落后、成本超支
2. 再写判断依据：`SV < 0`、`CV < 0`、`SPI < 1`、`CPI < 1`
3. 最后写趋势：`EAC > BAC`，若不纠偏将超预算
